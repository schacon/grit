# Grit v4 implementation plan — Complete plumbing

This plan covers **two goals**:

1. **Deepen test coverage** for the 23 plumbing commands already implemented (v1+v2)
2. **Add the remaining plumbing commands** from upstream Git

Same methodology as v1–v3: study upstream C source, port tests from `git/t/`, implement until tests pass.

**Checkbox legend:** `[ ]` not started · `[~]` claimed · `[x]` done and tests green.

## Phase 2 — New plumbing manipulators

These commands don't exist yet and need to be built from scratch.

### 2.1 Core manipulators (high value — needed by porcelain)

- [ ] **apply** — Apply a patch to files and/or the index. Upstream: `git/builtin/apply.c` (huge — 5000+ lines). Tests: `t410*` (42 scripts, ~300 test cases). **Strategy:** Start with basic unified diff application, then add rename, binary, 3-way, and edge cases incrementally.

  - [ ] apply.2 — `--index` (apply to index too), `--cached` (index only)

  - [ ] apply.4 — `--reverse`, `-R`

  - [ ] apply.7 — `--3way` / threeway merge

- [ ] **pack-objects** — Create a packed archive of objects. Upstream: `git/builtin/pack-objects.c`. Tests: `t5300-pack-object.sh` + `t531*`–`t533*`. **Required by:** `repack`, `gc`, `fetch`, `push`.

- [ ] **index-pack** — Build pack index file for an existing packed archive. Upstream: `git/builtin/index-pack.c`. Tests: `t5300`, `t5302-pack-index.sh`.
  - [ ] index-pack.1 — Parse pack and build `.idx` file
  - [ ] index-pack.2 — `--verify`, `--stdin`
  - [ ] index-pack.3 — Resolve deltas, handle thin packs with `--fix-thin`

- [ ] **merge-file** — Three-way file merge. Upstream: `git/builtin/merge-file.c`, `git/ll-merge.c`. Tests: `t6403-merge-file.sh` (39 tests).
  - [ ] merge-file.1 — Basic 3-way merge
  - [ ] merge-file.2 — `--ours`, `--theirs`, `--union` strategies
  - [ ] merge-file.3 — Conflict markers, `-L` label
  - [ ] merge-file.4 — Port `t6403` tests

- [ ] **unpack-objects** — Unpack objects from a packed archive. Upstream: `git/builtin/unpack-objects.c`. Tests: `t5300`.

  - [ ] unpack-objects.3 — Port relevant tests

- [ ] **prune-packed** — Remove extra objects that are already in pack files. Upstream: `git/builtin/prune-packed.c`.
  - [ ] prune-packed.1 — Remove loose objects present in packs
  - [ ] prune-packed.2 — `-n` dry run, `-q` quiet

## Completion criteria

v4 is done when:

1. All **existing** plumbing commands reach ≥50% upstream test coverage (Phase 1).
2. The **core new plumbing** commands are implemented: `apply`, `pack-objects`, `index-pack`, `merge-file`, `unpack-objects`, `diff-tree`, `diff-files`, `ls-remote` (Phase 2.1 + 3.1).
3. `cargo test --workspace` and `./tests/harness/run.sh` green.
4. `docs/index.html` regenerated showing updated coverage.
5. All work committed and pushed.

**Stretch goal:** Reach 200+ passing test scripts (currently ~60).
