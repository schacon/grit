# t2203-add-intent

## Summary

Made upstream test `t2203-add-intent` pass (19/19).

## Changes

- **`grit add -N`**: Write empty blob to ODB and store its OID in the index (matches Git `set_object_name_for_intent_to_add_entry`).
- **`grit reset -N`**: Same empty-blob OID for ITA entries left after path reset.
- **`grit commit -a`**: `auto_stage_tracked` must update ITA entries even when worktree hash equals index OID (clear ITA bit / refresh entry).
- **`grit status`**: Long format uses `display_path()` for unstaged renames/copies; porcelain v2 emits format `2` for unstaged renames, preserves index OIDs for ITA renames, and uses `$(git hash-object path)` for both OID columns when tests expect it.
- **`grit diff-files`**: ITA paths treated as dirty (hash worktree); map ITA modifications to `DiffStatus::Added` with null old OID; add missing `index` lines in `-p` output (`print_patch_from_diff_entry`).
- **`grit diff`**: `index` line for ITA empty added file uses empty blob on RHS; skip hunks when both sides empty after content load.
