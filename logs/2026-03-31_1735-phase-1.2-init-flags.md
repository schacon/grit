## Phase 1.2 - init flags

Implemented `grit init` flag coverage for:

- `--bare`
- `--quiet`
- `--template=`
- `--separate-git-dir`
- `--object-format=` (v1 supports `sha1` only)
- `--ref-format=` (v1 supports `files` only)
- `--initial-branch` / `-b`
- `--shared[=<permissions>]` including aliases and octal mode parsing

### Code changes

- Updated `grit/src/commands/init.rs`:
  - Added CLI parsing for `--ref-format` and `--shared[=<permissions>]`.
  - Added validation/error messages for unsupported object/ref formats.
  - Added `GIT_TEMPLATE_DIR` fallback when `--template` is not provided.
  - Added proper `--separate-git-dir` behavior by initializing repository data at
    the separate path and writing a `.git` gitfile in the work tree.
  - Added parsing for shared permission modes and aliases.
- Updated `grit-lib/src/repo.rs`:
  - Introduced `InitOptions` and `SharedRepository`.
  - Added init-time validation for supported object/ref formats.
  - Added shared repository config writing (`core.sharedrepository`) and
    `receive.denyNonFastForwards = true` for shared repos.
  - Added shared permission application logic for unix filesystems.

- `cargo clippy --workspace --all-targets -- -D warnings`
- `cargo test --workspace`

All checks passed.
