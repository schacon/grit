### Investigation Notes
- Direct test reproduction (`tests/t4006-diff-mode.sh`) reports failures in:
  - `--stat output after binary chmod`
  - `--shortstat output after binary chmod`
- In isolated upstream harness:
  - `bash scripts/run-upstream-tests.sh t4006-diff-mode` reports `7/7` passing.
- Root cause of apparent mismatch:
  - `test_chmod` in local `tests/test-lib.sh` uses `git update-index --chmod=...` only.
  - This modifies index mode but **does not change worktree executable bits**.
  - For `git diff HEAD` (tree vs worktree), binary file `binbin` then appears unchanged, so only `rezrov` appears in stat.
  - This explains local 5/7 while upstream (with native Git test harness behavior) passes.
