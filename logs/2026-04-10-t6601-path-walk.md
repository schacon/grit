# t6601-path-walk

## Fixes

- **`date_order_walk`**: Seed the heap with every selected commit whose in-degree from the selected subgraph is zero (Git `commit_list_insert_by_date` behavior). Previously every commit in the `tips` slice was seeded, which let inner commits run before descendants when `tips` was the full set and broke batch ordering.
- **Index + path-walk**: Git’s `setup_pending_objects` only calls `add_path_to_list` for index blobs/trees; it does not `push_to_stack` until the second `strmap_for_each_entry` pass. Added `add_path_pending` so index-sourced paths are not enqueued during `setup_pending_objects`, matching commit-root-tree ordering for `--indexed-objects --branches` (t6601.4).
- **`tests/test-tool`**: Route `path-walk` to `GUST_BIN` so `test-tool -C <repo> path-walk` does not fall through to system `git test-tool` (t6601.15 in isolated repos).
