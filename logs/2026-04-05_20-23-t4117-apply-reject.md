### Findings so far
- `grit apply` currently rejects `--reject` as an unknown argument.
- Even with partial-apply behavior from current hunk scanning, reject-mode expectations are not met:
  - expected output file updates do not match test fixture output for reject runs,
  - `.rej` file generation semantics are not implemented.
- Required next implementation area is explicit `--reject` support in `grit apply`:
  - parse and accept flag,
  - continue applying successful hunks,
  - emit reject hunks to `<path>.rej`,
  - return non-zero when any hunk rejects.

### Fix implemented
- Updated `grit/src/commands/apply.rs`:
  - Added `--reject` CLI flag to `apply` argument parsing.
  - Added `apply_hunks_with_reject` to apply each hunk independently and collect rejected hunks instead of hard-failing at first mismatch.
  - Added reject file rendering/writing (`<path>.rej`) with unified hunk headers and rejected hunk bodies.
  - Integrated reject-mode path in worktree apply flow:
    - successful hunks are written to target file,
    - rejected hunks are emitted to reject file,
    - command exits non-zero (`patch failed`) when any hunk is rejected.
