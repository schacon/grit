# Phase 4.2 reconcile gap

Goal: close the remaining `update-index` behavior gaps called out in `logs/2026-03-31_1957-plan-reconcile-gap.md` for `plan.md` item 4.2.

## Implemented

- Wired `--info-only` in `grit/src/commands/update_index.rs` to hash blob data without writing objects to ODB.
- Implemented `--again` to re-process tracked stage-0 entries, with:
  - default scope based on current working directory,
  - optional path selection from command arguments,
  - `--remove --again` removal of tracked-but-missing files,
  - fail-fast behavior when tracked paths are missing and `--remove` is not set.
- Improved refresh semantics:
  - `--refresh` and `--really-refresh` now use stat-diff checks,
  - `--really-refresh` includes `assume-unchanged` entries while `--refresh` skips them,
  - racy timestamp detection marks the index dirty so a rewrite breaks racily-clean state.
- Preserved entry flag bits when replacing existing index entries.
- Wrote the index only when there are actual changes (instead of rewriting unconditionally).
