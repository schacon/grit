## Implementation (complete)

### `grit apply` symlink correctness fixes (`grit/src/commands/apply.rs`)
- Parsed optional file mode from patch `index <old>..<new> <mode>` headers so normal modify patches (including symlink mode `120000`) carry mode information in `FilePatch`.
- Added `FilePatch::is_metadata_only_rename_or_copy()` and skip those entries during worktree apply, preventing accidental source-path deletion/rewrite for rename-only metadata sections.
- Tightened symlink escape overlay logic:
  - metadata-only renames/copies now propagate symlink-ness into the target overlay,
  - writes beyond symlink prefixes now emit Git-compatible create-path error format:
    `affected file '<path>' is beyond a symbolic link`.
- Precheck existence snapshot now treats symlinks as existing (`symlink_metadata`) so dangling/tracked symlink paths are not misclassified as missing.
- Missing-preimage fallback for `can_apply_with_empty_preimage()` now only applies when patch old OID is absent; otherwise missing preimages error out as expected.
- Missing preimage errors for path traversal cases now normalize to:
  `<path>: No such file or directory`.
- `--reject` completion error changed to `Rejected hunk` (Git-compatible grep target in test 8).
- Reject writer now removes existing symlink at `<path>.rej` before creating reject file.

### `grit diff-files` symlink patch/header fixes (`grit/src/commands/diff_files.rs`)
- Added `new_oid` to internal `Change` records and populated it from worktree hashes.
- For patch output (`-p`), emit `index <old>..<new> <mode>` header when mode unchanged, matching upstream patch format required by test 2.
- Switched patch/stat content readers to symlink-aware byte loading (read symlink target bytes instead of dereferencing).

### `grit clean` tracked symlink preservation (`grit/src/commands/clean.rs`)
- Directory walk now uses `symlink_metadata` + file-type checks, treating symlink-to-directory entries as directory-like for recursion/pathspec decisions.
- This prevents `clean -dfx` from deleting tracked symlink paths that have tracked entries under the linked directory prefix (critical for t4115 tests 6/7 setup).
