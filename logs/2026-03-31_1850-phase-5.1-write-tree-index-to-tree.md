## Phase 5.1 - write-tree from index

- Re-implemented index-to-tree construction in `grit-lib` (`grit-lib/src/write_tree.rs`) so the logic is not tied to the CLI layer.
- Build logic now:
  - filters stage-0 index entries,
  - recursively groups paths into subtrees,
  - sorts tree entries with Git tree ordering (`tree_entry_cmp`),
  - canonicalizes blob-like modes to Git tree expectations (`100644`, `100755`, `120000`; preserves gitlink `160000`),
  - writes tree objects through `Odb::write(ObjectKind::Tree, ...)`.
- Kept `grit write-tree` wired via `grit/src/commands/write_tree.rs`, delegating to the new library function.
- Added `grit-lib` unit test `writes_sorted_tree_with_canonical_modes` to verify sorting and mode behavior end-to-end by reading written tree objects back from the ODB.
- Validation:
  - `cargo fmt`
  - `cargo clippy --workspace --all-targets -- -D warnings`
  - `cargo test --workspace`
