- 10: `apply --directory -p (new file)`
- 11: `apply --directory (delete file)`

### Root causes found

1. `--directory` argument was used verbatim, so normalized-equivalent forms
   (`some/sub/`, `./some/sub`, `some//sub`, `some/./sub`, `some/../some/sub`)
   produced mismatched index/worktree paths (double slashes and dot segments).
2. Leading `../` in `--directory` was not rejected with the expected error.
3. Quoted C-style paths in patch headers (`diff --git "a/qu\157tefile" ...` and
   `+++ "b/qu\157tefile"`) were not decoded before path transforms, so apply
   attempted to stage the literal escaped path.

### Implementation

- `grit/src/commands/apply.rs`:
  - Added directory normalization helper for `--directory`:
    - collapses duplicate separators, `.` and internal `..`,
    - rejects leading `..` with `error: unable to normalize directory '<arg>'`.
  - Applied normalized directory value in `run_with_patch_input` for all
    path-transform, precheck, worktree, and index application flows.
  - Added C-style quoted path decoding for patch path parsing:
    - decode `diff --git` old/new paths,
    - decode `---` / `+++` header paths,
    - supports octal escapes used in upstream quoted filename fixture.
