## Phase 2.3 - hash-object --literally and stdin modes

- Implemented object payload validation in `gust hash-object` for non-literal hashing:
  - `-t tree` now validates parsed tree structure before hashing.
  - `-t commit` now rejects malformed commit payloads with a corrupt-commit error.
  - `-t tag` now validates required tag headers (`object`, `type`, `tag`).
- Wired `--literally` to bypass these validations so malformed objects can still be hashed/written when explicitly requested.
- Kept stdin processing order behavior intact (`--stdin` content is consumed first, then positional files) and added focused CLI/unit coverage for:
  - duplicate `--stdin` rejection
  - corrupt commit/tag rejection without `--literally`
  - corrupt commit acceptance with `--literally`.
- Updated `plan.md` to mark only item `2.3` as complete.
