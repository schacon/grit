# t4103-apply-binary

## Summary

Made `tests/t4103-apply-binary.sh` pass 24/24 against grit.

## Changes

- **`grit apply` (`apply.rs`)**
  - Parse `Binary files … differ` / `Files … differ` as binary metadata-only patches; resolve postimage from ODB when `index` has full 40-char OIDs (Git parity).
  - Strip `a/`/`b/` and apply `-p` after `normalize_mismatched_diff_git_paths`.
  - `GIT binary patch`: Git `encode_85` byte order, `literal`/`delta` parsing, no bogus newline after zlib inflate for deltas.
  - Full-null old OID (`000…40`) matches empty preimage for new files.
  - Binary apply: raw disk bytes when content has NUL (skip CRLF clean for preimage hash).
  - Skip text `verify_old_oid_matches_content` when patch is binary (CRLF text can disagree).
  - Truncated literal: report `corrupt binary patch at file:N: ` at EOF line like Git.
  - Unrecognized binary + empty patch: stderr lines matching Git; `SilentNonZeroExit` to avoid duplicate `error:` prefix.

- **`grit diff-tree` (`diff_tree.rs`)**
  - `--no-prefix` for path-limited `--binary` tests.
  - `--binary`: emit `GIT binary patch` with zlib + Git base85; full `index` OIDs when emitting binary; delta when `delta_encode` succeeds else literal + reverse `literal 0`.
