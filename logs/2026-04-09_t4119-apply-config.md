# t4119-apply-config

## Fixes

1. **`apply.whitespace=strip`** — Treat `strip` like Git’s `--whitespace=strip` / `fix` when reading config (not only `fix`).

2. **Work tree path resolution** — Git’s `setup_git_directory` changes to the repo root; patch paths are relative to that root. Resolve filesystem paths as `work_tree.join(rel)` so running `apply` from a subdirectory still updates `sub/file1` correctly.

3. **`is_toplevel_relative` (Git `prefix_patch`)** — `diff --git` patches keep paths as in the patch; traditional `---`/`+++` patches get the setup prefix prepended (e.g. `file1` → `sub/file1` when run from `sub/`). Used for worktree IO, `.gitattributes` / ws rules, symlink checks, and precheck.

4. **`guess_p_value`** — Match Git’s traditional-patch `-p` inference using the setup prefix so `a/sub/file` from `sub/` yields `-p2` (t4119 depth-2 case).
