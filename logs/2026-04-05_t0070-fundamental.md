  - missing `test-tool pkt-line send-split-sideband`

### Implementation
- Added `run_test_tool_mktemp(rest)`:
  - accepts exactly one template path.
  - supports `XXXXXX` substitution and creates a unique file.
  - on failure prints `mktemp: <template>: <error>` to stderr and exits non-zero.
- Added `run_test_tool_regex(rest)`:
  - supports `--bug` (returns success),
  - supports optional `--silent`,
  - supports `<pattern> [<string>] [<options>]`,
  - supports flags `ICASE`, `NEWLINE`, and accepts `EXTENDED`, `NOTBOL`, `NOTEOL`, `STARTEND`.
  - uses Rust `regex::RegexBuilder`.
- Added pkt-line helper implementation:
  - `run_test_tool_pkt_line_pack_raw_stdin()`
  - `run_test_tool_pkt_line_send_split_sideband()`
  - `run_test_tool_pkt_line_receive_sideband()`
  - `run_test_tool_pkt_line_unpack_sideband(args)`
  - dispatcher `run_test_tool_pkt_line(rest)` for:
    - `pack` (existing)
    - `pack-raw-stdin`
    - `unpack` (existing)
    - `send-split-sideband`
    - `receive-sideband`
    - `unpack-sideband`
- Wired new test-tool subcommands in main dispatch:
  - `"mktemp" => run_test_tool_mktemp(rest)`
  - `"regex" => run_test_tool_regex(rest)`
  - `"pkt-line" => run_test_tool_pkt_line(rest)`

### Follow-up and final status
- Root cause for remaining failures: `tests/test-tool` had a local `pkt-line`
  case that did not support `send-split-sideband`, `receive-sideband`, or
  `unpack-sideband`, so the shell wrapper intercepted and failed before reaching
  the new Rust implementation.

  `git test-tool pkt-line "$subcmd" "$@"`.
