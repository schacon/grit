## t4258-am-quoted-cr

- Claimed from Diff plan to address mailbox CRLF quoting handling in `grit am`.
- Baseline:
  - `./scripts/run-tests.sh t4258-am-quoted-cr.sh` => 1/4.
  - `bash scripts/run-upstream-tests.sh t4258-am-quoted-cr` => 1/4.

### Root causes

1. `grit am` did not parse `--quoted-cr=<action>`, so `--quoted-cr=strip` failed argument parsing.
2. Base64-encoded mailbox payloads were not decoded; raw base64 text was treated as patch text.
3. Interactive mode (`-i`) was unsupported, and interactive continue/resolved prompts were missing.
4. CRLF handling around state serialization/deserialization and payload parsing produced mismatches:
   - warning path (`warn`) needed to preserve patch failure behavior while still surfacing quoted-CR warning,
   - strip path needed to normalize CRLF to LF for successful patch apply,
   - serialized patch parsing needed LF-based line splitting to avoid retaining `\r` in stored patch metadata.
5. Config fallback for `mailinfo.quotedCr` needed case-insensitive key handling in this parser path.

### Implementation

- Extended `grit am` args and flow:
  - Added `-i/--interactive` support.
  - Added `--quoted-cr=<action>` support.
  - Added interactive patch selection prompt before session creation.
  - Added interactive `--continue/--resolved` confirm prompt.
- Added quoted-CR policy handling:
  - `warn` (default): emit warning and preserve CR to trigger patch failure semantics where expected.
  - `strip`: remove CR and normalize line endings to LF for application.
  - `nowarn`: suppress warning without stripping.
- Added base64 transfer decoding for mbox payloads when `Content-Transfer-Encoding: base64` is present.
- Refactored mbox parsing to split decoded payload into message body + diff consistently, including `---` separator and signature terminator handling.
- Updated patch parsing path in `am` to use `split('\n')` rather than `lines()` for better newline fidelity.
- Fixed serialized patch deserialization to split on `\n` and preserve expected boundaries.
- Added case-insensitive `mailinfo.quotedCr` lookup via `ConfigSet::get_case_insensitive`.
