## Task: t1306-xdg-files (10/21 -> 21/21)

### Context
- Picked next highest-priority incomplete plan item: `t1306-xdg-files`.
- Initial verbose run showed 9 failures; dominant issues were:
  - wrong global config precedence in `git config` (`--get` and `--global --list` paths),
  - missing XDG-default handling for ignores/attributes,
  - harness behavior divergence causing local user config pollution and per-test cwd resets.

### Root causes identified
1. **Config command global path selection was write-oriented for reads**
   - `grit config --global` read logic only targeted `~/.gitconfig`, ignoring XDG global file loading semantics expected by tests.
   - Required split behavior:
     - **read**: prefer existing `~/.gitconfig`, else existing XDG, else fallback path.
     - **write**: if `~/.gitconfig` exists write there; otherwise write XDG when it exists.

2. **Ignore matcher only parsed local `.git/config`**
   - `core.excludesfile` lookup ignored global/XDG config sources.
   - Missing default fallback to `$XDG_CONFIG_HOME/git/ignore` (or `$HOME/.config/git/ignore`) when `core.excludesfile` unset.

3. **check-attr did not load global attributes**
   - Only `.gitattributes` and `.git/info/attributes` were considered.
   - Missing `core.attributesfile` and XDG default global attributes file support.

4. **Local test harness divergence created false negatives**
   - `tests/test-lib.sh` set local `user.name`/`user.email` in setup repository, which masked expected global `user.name` values in this file.
   - `test_expect_success` forcibly `cd`-reset each test block, breaking `t1306`'s intentional `cd git` carry-over.

### Code changes
1. **`grit/src/commands/config.rs`**
   - Added separate helpers:
     - `global_config_path_for_read()`
     - `global_config_path_for_write()`
     - `global_gitconfig_path()`
     - `global_xdg_config_path()`
   - Updated:
     - `resolve_config_file()` to use write helper for `--global`.
     - `load_config()` `--global` path to use read helper.
     - non-repo fallback read path to use read helper.

2. **`grit-lib/src/ignore.rs`**
   - Replaced manual `.git/config` parsing with layered config loading:
     - `ConfigSet::load(Some(&repo.git_dir), true)`
   - `core.excludesfile` now sourced from merged config.
   - Added XDG default fallback helper:
     - `default_global_ignore_path()`.
   - Switched path expansion to shared config path parser (`parse_path`).
   - Removed old ad-hoc parser/expander helpers no longer needed.

3. **`grit/src/commands/check_attr.rs`**
   - Added global attributes support:
     - loads `core.attributesfile` via `ConfigSet::load`.
     - falls back to `$XDG_CONFIG_HOME/git/attributes` then `$HOME/.config/git/attributes`.
   - Updated `load_gitattributes` signature to accept `Repository` and include global attrs first (lower precedence).

4. **`tests/test-lib.sh` (harness alignment)**
   - Removed automatic local identity writes in setup and `test_create_repo`.
   - Aligned `test_expect_success` execution behavior with upstream expectations by preserving working-directory changes across test blocks.
