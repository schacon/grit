# t1900-repo-info — `git repo info`

## Goal

Implement `grit repo info` to match upstream `git/builtin/repo.c` behavior exercised by `tests/t1900-repo-info.sh`.

## Changes

- `grit/src/commands/repo.rs`: `run_repo_info` parses `--format`, `-z`, `--all`, `--keys`, and positional keys; prints `layout.bare`, `layout.shallow` (presence of `.git/shallow`), `object.format` (`extensions.objectformat` in config, default `sha1`), `references.format` (`reftable` vs `files` via `reftable::is_reftable_repo`). Lines format uses C-style quoting for values when needed (control chars). Nul format: `key\nvalue\0` per field.
- Fatal cases (exit 128, `fatal:`): invalid `--format`, `--keys` with wrong format or with `--all`/keys, `--all` with keys, unsupported `table` format when not in keys-only mode.
- Partial success: stderr `error: key '…' not found` for bad keys, stdout still has valid keys; exit 1 without double-printing stderr via `SilentNonZeroExit`.
- `grit/src/explicit_exit.rs` + `main.rs`: `SilentNonZeroExit` handled before generic `error:` printing.
