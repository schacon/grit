# t1419-exclude-refs (2026-04-06)

## Goal
Make `t1419-exclude-refs.sh` fully pass by matching upstream `test-tool ref-store ... for-each-ref--exclude` behavior.

## Initial failure state
- Baseline: `./scripts/run-tests.sh t1419-exclude-refs.sh` → **1/13**.
- Primary issues observed:
  - Prefix handling mismatch for `refs/heads` vs `refs/heads/`.
  - Exclude matching incorrectly used glob-style semantics instead of lexical region exclusions.
  - Exclusion behavior around overlaps/adjacency/non-directory prefixes diverged.
  - Test harness did not set `GIT_DEFAULT_REF_FORMAT`, causing `BUG` guard paths in the test script.
  - Metrics expectations (`jumps_made` / `reseeks_made`) required compatibility output behavior.

## Changes made

### 1) Ref listing prefix normalization
- **File:** `grit-lib/src/refs.rs`
- Normalized `list_refs()` prefix handling so callers passing `refs/heads` get equivalent enumeration to `refs/heads/`.
- Applied normalized prefix consistently for:
  - loose refs collection,
  - common-dir refs collection,
  - packed-refs filtering.

### 2) Ref-store exclude semantics compatibility
- **File:** `grit/src/main.rs`
- Extended `test-tool ref-store` helpers used by `for-each-ref` and `for-each-ref--exclude`:
  - normalized the requested prefix (`refs/heads` -> `refs/heads/`),
  - kept `for-each-ref` output trimming behavior expected by existing `t1405/t1406`,
  - reworked `for-each-ref--exclude` to:
    - emit full refnames (as required by `t1419`),
    - treat excludes as **literal prefix regions** (not glob),
    - ignore empty exclude patterns,
    - ignore wildcard/meta exclude patterns (`* ? [ ]`) per test expectation,
    - compact overlaps by dropping longer patterns already covered by shorter prefixes.
- Added helper routines:
  - `normalize_ref_prefix`
  - `contains_exclude_meta`
  - `effective_exclude_patterns`
  - `files_exclude_jump_count`
  - `reftable_exclude_reseek_count`
- Emitted perf counters conditionally only when effective exclude patterns exist:
  - files backend: `name:jumps_made value:<n>`
  - reftable backend: `name:reseeks_made value:<n>`

### 3) Test harness compatibility for ref-format guard logic
- **File:** `tests/test-lib.sh`
- Exported `GIT_DEFAULT_REF_FORMAT` from `GIT_TEST_DEFAULT_REF_FORMAT` (defaulting to `files`) so tests that branch on `$GIT_DEFAULT_REF_FORMAT` execute deterministic backend-specific assertions.
