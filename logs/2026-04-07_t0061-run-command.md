# 2026-04-07 — t0061-run-command

## Goal

Make `t0061-run-command.sh` fully pass.

## Baseline

- Initial run: `GUST_BIN=/workspace/target/release/grit TEST_VERBOSE=1 bash t0061-run-command.sh`
- Result: **4/24** passing.
- Main failure: `test-tool: unknown subcommand 'run-command'` plus many missing run-command API semantics.

## Root causes

1. `test-tool run-command` helper was not implemented in `grit/src/main.rs`.
2. No Git-compatible command resolution behavior for:
   - slash vs PATH lookup,
   - executable vs directory/non-executable precedence,
   - ENOENT/EACCES-style fatal diagnostics.
3. No helper-mode behavior for parallel/stdin/abort/no-jobs test-tool paths.
4. Missing `GIT_TRACE` formatting equivalent to `trace: run_command: ...`.

## Implementation

Implemented a `test-tool run-command` compatibility helper in `grit/src/main.rs`:

- Added command resolution primitives:
  - `CommandResolutionError`,
  - `command_has_dir_sep`,
  - `is_executable_path`,
  - `resolve_command_path`.
- Added process-launch helpers:
  - `apply_env_operations`,
  - `run_command_capture` / `run_command_inherit`,
  - shell fallback (`/bin/sh`) for ENOEXEC-like script-without-shebang behavior.
- Added trace output helpers:
  - `render_run_command_trace_env`,
  - `emit_trace_run_command`.
- Added fatal diagnostics:
  - `emit_cannot_exec` producing `fatal: cannot exec '<cmd>': <reason>`.
- Implemented `run_test_tool_run_command(rest)` modes:
  - `inherited-handle`,
  - `start-command-ENOENT`,
  - `run-command`,
  - `run-command-parallel`,
  - `run-command-stdin`,
  - `run-command-abort`,
  - `run-command-no-jobs`.
- Wired dispatch:
  - in `test-tool` subcommand switch, added `"run-command" => run_test_tool_run_command(tool_args)`.
