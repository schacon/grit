### Initial observations
- Pathspec trailing-slash handling is too permissive for non-directory paths (`file0/` still matches `file0`).
- `diff-tree` wildcard pathspec handling is incomplete:
  - root tree resolution via `$EMPTY_TREE` (`4b825d...`) is failing with object lookup errors,
  - recursive/non-recursive wildcard matching behavior diverges from expected Git semantics.
- Remaining tests after these assertions (submodule + multi-pathspec) currently pass.

### Implementation notes
- `grit/src/commands/diff_index.rs`
  - tightened trailing-slash pathspec matching so `spec/` matches descendants only (`path.starts_with("{spec}/")`), not the file path itself; this fixes `file0/` false positives.
- `grit/src/commands/diff_tree.rs`
  - adjusted recursion implication logic so only `--patch` and `--stat` imply recursion; `--name-only`/`--name-status` now preserve non-recursive behavior unless `-r` is explicitly set.
  - improved pathspec filtering for tree-like entries:
    - wildcard specs can match directory entries when they match descendants (e.g. non-recursive `f*` and `path1/f*`),
    - trailing-slash specs only match directory-like entries.
  - added canonical empty-tree handling for upstream `$EMPTY_TREE`:
    - accept canonical empty tree hash (`4b825dc642cb6eb9a060e54bf8d69288fbee4904`) in `resolve_to_tree`,
    - normalize that OID to `None` before diff execution so empty-tree comparisons work without object lookup.
