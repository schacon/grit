# t3903-stash progress

## Summary

Improved `grit stash` and related plumbing toward `t3903-stash.sh`. After changes, harness reports **73/142** passing (was 57/142 at start of session).

## Key changes

- **rev_parse**: bare `stash` resolves to `refs/stash` (DWIM).
- **stash apply**: `--no-index` vs `stash.index` config; without `--index`, index reset to current HEAD for touched paths (not stash index merge); removed erroneous post-apply stat refresh that matched worktree into index.
- **build_index_from_tree**: populate blob `size` for stat cache (fixes spurious `diff-files` noise after `reset_to_head`).
- **stash show**: diff/stat against `stash^2` (index parent) like Git; `--numstat`; stricter unknown-option handling for `stash show`.
- **export/import**: `stash export` / `stash import` with validation roughly matching upstream chain format.
- **push**: `-mfoo` via `allow_hyphen_values`; `assume_push_or_error` for bare `stash drop` etc.; patch mode delegates to `checkout_patch`; intent-to-add-only rejection; `Save` gains `-p`.
- **drop/pop**: reject 40-char OIDs that exist as commits; `do_branch` drops stash only after successful apply.
- **main.rs**: invalid top-level stash options print upstream `or: git stash …` lines; `stash push -h` prints adoc push synopsis (`usage: git stash [push`).

## Remaining failures (examples)

Symlinks, directory/file switches, submodule stash message, index locking, `core.fsyncmethod`, skip-worktree, combined diff in `stash list --cc`, and other edge cases still need work.
