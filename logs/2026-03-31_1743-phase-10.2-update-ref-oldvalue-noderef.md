## Scope

- Completed only `plan.md` item 10.2.
- Focused files:
  - `gust/src/commands/update_ref.rs`
  - `plan.md`

## What changed

- Added old-value verification helper logic that treats all-zero oldvalue
  (`0000000000000000000000000000000000000000`) as "ref must not exist".
- Reused the same old-value verification path for single and stdin-driven
  `update`, `delete`, and `verify` flows.
- Added branch-name normalization for update targets so branch-like short names
  are treated as `refs/heads/<name>` while preserving full ref names and
  pseudorefs (`HEAD`, uppercase pseudoref names, or explicit `refs/...` names).
- Added `refs/heads/<name>` fallback when resolving `new-value` as a ref,
  matching callers that provide a short branch name instead of a full refname.
