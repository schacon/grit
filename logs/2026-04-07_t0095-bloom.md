## t0095-bloom — completion log (2026-04-07)

### Goal
Make `tests/t0095-bloom.sh` fully pass by implementing missing `test-tool bloom` support in `grit`.

### Baseline
- `./scripts/run-tests.sh t0095-bloom.sh` → **1/11**.
- Verbose failures showed every assertion failing with:
  - `error: test-tool: unknown subcommand 'bloom'`

### Root cause
`grit/src/main.rs` did not dispatch a `test-tool bloom` subcommand, and there was no compatibility implementation for:
- `get_murmur3`
- `get_murmur3_seven_highbit`
- `generate_filter`
- `get_filter_for_commit`

### Implementation
Added `test-tool bloom` support in `grit/src/main.rs`:

1. **Dispatch**
- Added `"bloom" => run_test_tool_bloom(tool_args)` in `test-tool` command routing.

2. **Murmur3 helper parity**
- Implemented Murmur3 v2 and v1 seeded hash functions matching upstream helper semantics.
- Added `get_murmur3` and `get_murmur3_seven_highbit` output format:
  - `Murmur3 Hash with seed=0:0x........`

3. **Bloom key/filter generation**
- Added constants matching upstream defaults used in t0095:
  - `num_hashes = 7`
  - `bits_per_entry = 10`
  - `max_changed_paths = 512`
- Implemented key derivation with upstream seeds (`0x293ae76f`, `0x7e646e2c`) and additive hash sequence.
- Implemented bit placement in byte filter and exact output formatting:
  - `Hashes:...|`
  - `Filter_Length:<n>`
  - `Filter_Data:<hex>|...|`

4. **Commit filter computation**
- Implemented `get_filter_for_commit` by:
  - resolving commit and first parent tree,
  - computing first-parent tree diff via `grit_lib::diff::diff_trees`,
  - collecting changed paths plus all leading directory prefixes,
  - truncating to `ff` if changed path count exceeds 512,
  - generating 1-byte zero filter for empty change set.
