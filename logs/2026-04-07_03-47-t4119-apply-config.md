### Implementation summary
- Added explicit `--unsafe-paths` option parsing to `apply` args and wired `am`’s internal `apply` invocation to initialize the new field.
- Extended whitespace mode handling so `--whitespace=strip` (and `apply.whitespace=strip`) is treated as a whitespace-fix mode.
- Added robust traditional unified-diff header parsing:
  - strip timestamps/trailing metadata from `---`/`+++` tokens,
  - decode quoted paths,
  - handle `a/`/`b/` prefixes and `file+` cases from GNU-style patches.
- Fixed subdirectory path handling by computing a worktree-relative prefix from current cwd and applying/removing it consistently when mapping patch paths.
- Added per-path whitespace attribute support for apply:
  - extended `grit-lib` `FileAttrs` with `whitespace_blank_at_eol`,
  - parse `whitespace=blank-at-eol` / `whitespace=-blank-at-eol` from `.gitattributes`,
  - in `apply`, disable whitespace stripping for patch paths where attr resolves to `-blank-at-eol`.
- Updated index/check/worktree apply paths to use attribute-aware whitespace mode resolution.
