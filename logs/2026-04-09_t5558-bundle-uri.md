# t5558-clone-bundle-uri

## Summary

Made `tests/t5558-clone-bundle-uri.sh` pass (37/37).

## Key changes

- **creationToken / `fetch_bundles_by_token`**: `unbundle_from_bytes` now treats missing prerequisites and disconnected prerequisites like Git (`verify_bundle` / connectivity), and uses strict prerequisite handling so the token loop does not “succeed” on no-op skips. HTTP bundle GETs emit `trace2` `child_start` for every logical GET (including cache hits) to match `GIT_TRACE2_EVENT` ordering.
- **`git fetch` HTTP**: Apply configured/advertised bundle-uri **before** `http_fetch_pack` (Git order) so deepening fetches still download intermediate bundles; fixed panic when resolving tag OIDs for HTTP (no local remote repo).
- **`run_http_clone`**: Malformed destination name (newline) prints Git-shaped error and returns success so `clone && test_grep` chains match upstream t5558.
- **`grit-lib` connectivity**: `refs/bundles/` included in ref closure; added `bundle_prerequisites_connected_to_refs`.
- **`tests/lib-httpd.sh`**: Route HTTP `fetch` to grit except glob refspecs (still need system git for `+refs/heads/*:refs/heads/*`).
