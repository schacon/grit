# 2026-04-01 16:33 — Config System (v3 Phase 0.1)

## Task
Implement `gust config` — Git-compatible configuration file reading and writing.

## What was built

### Library (`gust-lib/src/config.rs`)
- **Parser**: Full Git config format — sections, subsections with quoted strings, key=value pairs, bare boolean keys, inline comments (#/;), escape sequences (\n, \t, \\, \"), quoted values
- **ConfigFile**: Single-file operations — parse, set, unset, remove-section, rename-section, round-trip write
- **ConfigSet**: Multi-file merge with cascade (system → global → local → worktree → command)
- **Type coercion**: `parse_bool` (true/yes/on/1, false/no/off/0), `parse_i64` (k/m/g suffixes), `parse_path` (~/ expansion)
- **Include directives**: `[include] path = ...` processed; `[includeIf]` conditions stubbed (returns false — safe default)
- **Environment**: `GIT_CONFIG`, `GIT_CONFIG_GLOBAL`, `GIT_CONFIG_COUNT`/`KEY_N`/`VALUE_N`

### CLI (`gust/src/commands/config.rs`)
- Modern subcommands: `get`, `set`, `unset`, `list`, `rename-section`, `remove-section`, `edit`
- Legacy flags: `--get`, `--get-all`, `--get-regexp`, `--unset`, `--unset-all`, `--list`, `--rename-section`, `--remove-section`, `--replace-all`
- File scope: `--system`, `--global`, `--local`, `--worktree`, `--file`
- Type: `--bool`, `--int`, `--path`, `--type`
- Display: `--show-scope`, `--show-origin`, `--name-only`, `-z`

### Tests (`tests/t1300-config.sh`)
25 tests covering: get/set/unset (legacy + subcommand), overwrite, mixed case, subsections, special characters, --bool, --int, --list, --local, remove-section, rename-section, --show-scope, --name-only, -z, get --all.
