# 2026-04-06 — t0100-previous

## Scope
- Target file: `tests/t0100-previous.sh`
- Starting status: `2/6` passing
- Goal: make `@{-N}` previous-branch behavior fully compatible for branch deletion, merge naming, and reflog walking.

## Failures addressed
1. `git branch -d @{-1}` failed because branch deletion treated `@{-N}` as literal branch names.
2. Merge commit messages used raw `@{-1}` expressions instead of resolved branch names (`Merge branch 'other'` expected).
3. `git log -g --format=%gd @{-1}` selected wrong reflog namespace and produced extra entries.

## Implemented fixes

### `grit/src/commands/branch.rs`
- Updated `delete_branch()` to resolve user branch arguments via `symbolic_full_name()` and accept only local `refs/heads/*` targets.
- This allows `branch -d @{-1}` to delete the actual previous branch instead of failing lookup.

### `grit/src/commands/merge.rs`
- Added merge target display normalization:
  - `merge_target_display_name()`
  - `short_ref_display_name()`
- `build_merge_message()` now uses resolved symbolic refs (including `@{-N}`) to produce Git-compatible human-facing messages.
- This fixes message expectations for `merge @{-1}` and `merge @{-1}~1`.

### `grit/src/commands/log.rs`
- In reflog walk mode (`log -g`), ref selection now first resolves revision arguments with `symbolic_full_name()`.
- This correctly maps `@{-1}` to its branch ref before reading reflogs.

### `grit/src/commands/checkout.rs`
- Adjusted `write_checkout_reflog()` to update only `HEAD` reflog, not branch reflogs, matching native checkout behavior.
- Prevents unrelated branch reflog noise that caused `log -g @{-1}` to show extra `%gd` entries.
