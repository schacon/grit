## Task: t4217-log-limit

### Baseline
- `./scripts/run-tests.sh t4217-log-limit.sh` initially reported `1/3` passing (`2` left).
- Failing cases:
  - `git log --since-as-filter=...`
  - `git log --children --since-as-filter=...`

### Root cause
- Test fixture commits set `GIT_COMMITTER_DATE` in human form (`YYYY-MM-DD HH:MM`), not epoch.
- `grit log` date filtering read committer timestamps via `extract_epoch_from_ident`, which only parsed epoch-form idents and returned `0` for human-form dates.
- As a result, all commits were filtered out by `--since-as-filter`.

### Fix implemented
- Updated `grit/src/commands/log.rs`:
  - Added robust committer-date parsing in `extract_epoch_from_ident`:
    - fast path for epoch format (`<epoch> <tz>`)
    - fallback for human-readable date strings via `parse_date_to_epoch`.
  - Improved `parse_date_to_epoch`:
    - date-only (`YYYY-MM-DD`) now maps to end-of-day (`23:59:59`) so the date is inclusive, matching Git behavior expected by tests.
