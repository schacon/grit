## t4046-diff-unmerged

### Baseline
- `./scripts/run-tests.sh t4046-diff-unmerged.sh` -> 1/8 passing.
- Direct run (`bash tests/t4046-diff-unmerged.sh` with `GUST_BIN`) showed:
  - `diff-files -0` emitted unmerged raw lines with `000000` new mode instead of expected `100644`.
  - `diff-files -1/-2/-3` omitted required `U` rows and only emitted stage-specific `M` rows for matching paths.
  - `git diff --cached --stat` for unmerged index produced no `Unmerged` lines.
  - `git diff --cached --quiet` (with and without `--ignore-all-space`) returned exit code 0 instead of 1.

### Implementation summary
#### `grit/src/commands/diff_files.rs`
- Reworked unmerged collection logic in `collect_changes`:
  - Tracks preferred unmerged mode per path (`stage 2` preferred, then `3`, then `1`) for raw `U` output.
  - `stage == 0`: now always emits one `U` record per unmerged path (`:000000 100644 ... U\tpath`) matching upstream expectations.
  - `stage in 1..=3`: now emits both:
    - the unmerged `U` record for every unmerged path,
    - plus a stage-specific `M`/`D` comparison row for paths that carry the requested stage.
- Added stage fallback behavior:
  - bare `git diff-files` now defaults to stage 2 when unmerged entries exist (matching upstream default behavior in this test).

#### `grit-lib/src/diff.rs`
- Updated `diff_index_to_tree` to surface unresolved index stages as staged diffs:
  - detects any path with non-zero-stage entries and no stage-0 entry.
  - emits `DiffStatus::Unmerged` entries with:
    - `old_mode=000000`,
    - `new_mode` from preferred stage mode (2 > 3 > 1),
    - zero OIDs on both sides.
- This makes porcelain `git diff --cached` paths (`--stat`, `--quiet`) observe unmerged entries and behave like upstream.
