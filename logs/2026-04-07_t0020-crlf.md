# t0020-crlf — 2026-04-07

## Goal

Make `tests/t0020-crlf.sh` fully pass by aligning CRLF conversion and safety behavior with upstream Git across add/update/checkout/apply/rm paths.

## Baseline

- Initial harness run:
  - `./scripts/run-tests.sh t0020-crlf.sh`
  - Result: **16/36** passing.
- Early failing clusters:
  - safecrlf mixed-line behavior,
  - read-tree/checkout-index file restoration and attribute-source handling,
  - apply `--index` content/index mismatch under autocrlf=true,
  - checkout/rm interactions around `.gitattributes` transitions.

## Root causes fixed

1. **safecrlf mixed handling mismatch**
   - `autocrlf=true` + `safecrlf=true|warn` mixed LF/CRLF files were not rejected/warned as expected.

2. **Index-side operations skipped clean conversion**
   - `update-index` and `apply --index` hashed raw worktree/patch content rather than clean-converted content, causing false index mismatch/failure.

3. **Checkout/read-tree conversion source for attrs**
   - checkout/read-tree/checkout-index behavior depended too much on current worktree `.gitattributes` availability/order.

4. **Dirty/safety checks compared only one representation**
   - checkout/rm safety checks could classify conversion-equivalent content as dirty in `.gitattributes` transition scenarios.

## Implementation summary

- `grit-lib/src/crlf.rs`
  - updated safecrlf input checks to treat mixed LF/CRLF as unsafe for `autocrlf=true` in safe/warn modes.
  - preserved mixed-file checkout behavior for `autocrlf=true` (do not force-normalize mixed files on checkout path).
  - added support for legacy `.gitattributes` `crlf` attribute forms used by upstream tests.

- `grit/src/commands/update_index.rs`
  - applied clean conversion (`convert_to_git`) before object hashing for regular files.

- `grit/src/commands/apply.rs`
  - made `--index` verification and index write paths conversion-aware (clean-side hashing).
  - added `.gitattributes` self-conversion guard in apply index checks/writes.

- `grit/src/commands/read_tree.rs`
  - made overwrite/dirty checks conversion-aware (`convert_to_git`) with robust `.gitattributes` handling.

- `grit/src/commands/checkout_index.rs`
  - fixed leading-directory creation behavior so `checkout-index -u one dir/two three` restores nested paths correctly.

- `grit/src/commands/checkout.rs`
  - made dirty checks conversion-aware and tolerant of raw/normalized equivalence for `.gitattributes`.
  - made worktree update logic attribute-source robust for the target index during branch/tree transitions.
  - removed stale conversion fallback that could force incorrect attr interpretation in transition cases.

- `grit/src/commands/rm.rs`
  - made safety checks conversion-aware using clean-side comparison against index OIDs.

- `grit/src/commands/diff_files.rs`
  - aligned diff-files file-hash comparison with conversion-aware behavior used by this compatibility path.
