# t8005-blame-i18n

- Extended `parse_commit` to accept non-UTF-8 author/committer header payloads; store raw bytes in `CommitData` (`author_raw`, `committer_raw`) and decode `message` using `encoding` header.
- `git commit` with `i18n.commitencoding` now encodes author, committer, and message body for storage (matches Git object layout).
- `grit blame`: `--incremental`, `--encoding`, config-driven output encoding (`i18n.logoutputencoding` / `i18n.commitencoding`); porcelain metadata uses byte-accurate lines for `--encoding=none`.
- Moved encoding helpers to `grit-lib::commit_encoding`; `git_commit_encoding` re-exports for merge/rebase.

Validation: `./scripts/run-tests.sh t8005-blame-i18n.sh`, `cargo test -p grit-lib --lib`.
