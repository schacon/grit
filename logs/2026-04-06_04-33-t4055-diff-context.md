## Task: t4055-diff-context

### Claim
- Claimed from `PLAN.md` after completing `t4207-log-decoration-colors`.
- Baseline in tracked harness: `5/10` passing.

### Baseline failures
- `./scripts/run-tests.sh t4055-diff-context.sh` reported 5 failing assertions.
- Direct run from `tests/` showed:
  - `git log -U4 -1` rejected `-U` (`unexpected argument '-U'`).
  - `diff.context` config not applied to `git log -p` and `git diff`.
  - Invalid config handling mismatched expected failures:
    - non-numeric `diff.context=no` should fail with bad numeric config error.
    - negative `diff.context=-1` should fail with bad config variable error.

### Root causes
1. `log` command did not parse `-U/--unified`, so command-line context override was unavailable.
2. `log` patch generation hardcoded context lines to `3` instead of honoring config/CLI.
3. `diff` command ignored `diff.context` config entirely (always defaulted to `3` unless `-U` set).
4. `diff` config parsing accepted invalid `diff.context` values silently instead of failing.

### Implementation
- `grit/src/commands/log.rs`:
  - Added `-U/--unified` argument (`pub unified: Option<usize>`) to log args.
  - Added `resolve_patch_context_lines()` and `parse_diff_context_config_value()`:
    - precedence: CLI `-U` > `diff.context` config > default `3`.
    - error behavior for config now matches expected classes:
      - non-numeric → `fatal: bad numeric config value ... invalid unit`
      - negative → `fatal: bad config variable 'diff.context'`
  - Wired resolved context into both walk and no-walk diff emission paths.
  - Updated patch writer call chain to pass context parameter through `write_commit_diff`.

- `grit/src/commands/diff.rs`:
  - Added `resolve_diff_context_lines()` and reused `parse_diff_context_config_value()` logic.
  - Context resolution now follows:
    - CLI `--unified/-U` > `diff.context` config > default `3`.
  - Invalid `diff.context` config now fails fast with git-compatible error classes/messages.
