## Phase 4.2 update-index

- Reviewed `plan.md` scope for `4.2` and audited `gust/src/commands/update_index.rs`.
- Implemented `update-index` behavior improvements required by this phase:
  - Enforced `--add` for adding previously untracked paths.
  - Added stricter remove semantics for `--remove` vs `--force-remove`.
  - Implemented `--info-only` object hashing path (no object write) where content is updated.
  - Expanded `--index-info` metadata parsing to support:
    - `<mode> <oid>`
    - `<mode> <type> <oid>`
    - `<mode> <oid> <stage>`
  - Added stage-aware flag encoding for `--index-info` records.
  - Improved refresh behavior:
    - Skip assume-unchanged entries for normal refresh.
    - Respect `--really-refresh` to include assume-unchanged entries.
    - Respect `--again` by updating only entries whose cached stat differs.
    - Refresh full stat tuple (`ctime`, `mtime`, `dev`, `ino`, `mode`, `uid`, `gid`, `size`).
- Added focused unit tests in `gust/src/commands/update_index.rs` for `--index-info` parsing and stage handling.
- Marked only `plan.md` item `4.2` as complete (`[x]`).
