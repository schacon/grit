# t4151-am-abort (2026-04-08)

## Goal

Make `tests/t4151-am-abort.sh` (20 cases) pass against `grit`.

## Changes (summary)

- **`grit/src/commands/am.rs`**
  - `apply_hunks`: fail on EOF when patch expects context/remove lines (fixes silent wrong applies, e.g. patch 5 after skipping 4).
  - Session files: `abort-safety`, `ORIG_HEAD` at start only (not after each commit), `dirtyindex` when index ≠ HEAD at apply start.
  - `am --skip`: `rerere` clear + `read_tree::am_clean_index(HEAD, HEAD)` then continue.
  - `am --abort`: `safe_to_abort`, `rerere` clear, `am_clean_index` + restore HEAD from `orig-head`/`head-name`, remove `ORIG_HEAD`, exit **128** if clean fails.
  - `--3way`: strict preimage check when applying patch; whole-file 3-way rules; unborn HEAD uses empty tree; modify/delete conflicts (stages 1+3 only); resolve patch index lines with `resolve_revision_for_patch_old_blob`.
- **`grit/src/commands/read_tree.rs`**
  - `am_clean_index`: Git `clean_index` analogue (reset index to HEAD tree preserving stats, two-way merge toward `ORIG_HEAD`, checkout; materialize phase1 before phase2 for `require_uptodate`).
  - `checkout_index_entries`: if target is a directory with untracked contents, error like Git (“would lose untracked files”).
