## t4139-apply-escape

### Baseline
- Claimed task in `PLAN.md` (`[~]`).
- Local baseline: `./scripts/run-tests.sh t4139-apply-escape.sh` → **8/12**.
- Upstream baseline: `bash scripts/run-upstream-tests.sh t4139-apply-escape` → **6/12**.
- Primary failing behavior: `git apply` accepted escaping paths like `../foo` even without `--unsafe-paths`, and also in `--index`/`--check` modes.

### Root cause
- `apply::Args` defined `unsafe_paths` but the clap attribute was accidentally removed, so `--unsafe-paths` was not parsed as an option.
- Safety checks therefore did not align with upstream expectations for explicit/implicit unsafe-path handling.

### Implementation
- `grit/src/commands/apply.rs`
  - Restored CLI flag parsing:
    - `#[arg(long = "unsafe-paths")] pub unsafe_paths: bool`
  - Kept repository safety enforcement logic:
    - reject path escape (`..` / absolute) by default.
    - allow escapes only when `--unsafe-paths` is set **and** neither `--index` nor `--cached` is active.
- `grit/src/commands/am.rs`
  - Updated in-memory `apply_command::Args` construction to set `unsafe_paths: false` explicitly after struct expansion.
- Clean-up:
  - Removed now-unused legacy unsafe-path helper functions in `apply.rs`.
