## Implementation + completion

### Core fixes implemented

- `fsck`:
  - Added commit header validation in `grit/src/commands/fsck.rs`:
    - validates author/committer identity header shape (`Name <email> <ts> <tz>`),
    - reports broken author headers as `invalid.author` (matching t4212 expectation),
    - keeps existing parse-based structural validation.

- `log` invalid-corrupt metadata handling in `grit/src/commands/log.rs`:
  - Added robust ident-date parsing with explicit states:
    - valid date,
    - numeric overflow sentinel,
    - invalid/missing/whitespace date.
  - `%ad`/date placeholders now follow expected sentinel behavior:
    - overflow numeric dates render as epoch-style date (`Thu Jan 1 00:00:00 1970 +0000`),
    - non-numeric/whitespace invalid dates render as empty string for placeholders.
  - header Date rendering (`git log` default/medium/full/fuller/email) now uses header-specific fallback:
    - invalid/overflow dates render epoch sentinel header date instead of raw broken tokens.
  - adjusted pretty-print separator logic to avoid extra trailing blank line in default format,
    matching upstream output shape in broken-email case.

- `rev-list` `--until/--before/--since/--after` support in `grit/src/commands/rev_list.rs`:
  - Added option parsing and filtering for date-limited traversal.
  - Implemented commit timestamp parsing with sentinel semantics:
    - valid numeric timestamp: parsed normally,
    - overflow numeric timestamp: sentinel epoch (`0`),
    - invalid/whitespace timestamp: sentinel epoch (`0`).
  - Ensures whitespace-only committer dates and overflow/bad dates behave as expected under `--until`.
