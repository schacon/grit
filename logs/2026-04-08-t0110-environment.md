# t0110-environment

## Issue

`t0110-environment.sh` failed after the first test because the harness shell stayed in `repo/`; the next test ran `cd repo` from there and looked for `repo/repo`.

## Fix

In `tests/test-lib.sh`, `test_run_` now runs `cd "$TRASH_DIRECTORY"` before each test body (same assumption as upstream Git’s test-lib: each test starts at trash root).

## Verification

`./scripts/run-tests.sh t0110-environment.sh` → 31/31 pass.
