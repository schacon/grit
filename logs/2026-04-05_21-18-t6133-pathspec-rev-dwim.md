    - `^{foo} with metacharacters dwims to rev` (`git log "HEAD^{/b.*}"`)
    - `@{foo} with metacharacters dwims to rev` (`git log "HEAD@{now [or thereabouts]}"`)
    - `:/*.t from a subdir dwims to a pathspec` (`git log ":/*.t"`)

### Root causes
1. **`log` treated some non-revision tokens as hard revision errors**
   - In `grit/src/commands/log.rs`, revision parsing aborted on unknown tokens even when they were likely pathspecs (wildcards and magic pathspec syntax) and no `--` separator was provided.
2. **`rev-parse` lacked support for `^{/pattern}` peel search**
   - `grit-lib/src/rev_parse.rs` parsed `^{...}` but treated `{/...}` as unsupported.
3. **`@{<date-ish>}` parsing was too strict**
   - `approxidate()` only recognized date-like strings with `YYYY-MM-DD`; it did not handle `now` with extra text (`now [or thereabouts]`).
4. **`^{/pattern}` search did exact substring matching**
   - Needed regex-like matching semantics so `b.*` would match commit message `brackets`.

### Fixes
1. **Pathspec DWIM in `log`** (`grit/src/commands/log.rs`)
   - Added heuristic `is_likely_pathspec_during_rev_parse(token)`:
     - Treat token as pathspec candidate when unknown revision and token has wildcard/magic pathspec markers, while excluding clear revision syntax (`^{`, `@{`, `..`).
   - During revision collection:
     - unknown-revision + pathspec-like token → collect into `implied_pathspecs` instead of failing.
   - Combined explicit `args.pathspecs` + `implied_pathspecs` for commit filtering and `--follow`.
   - Reused existing pathspec scope validation for implied pathspecs.

2. **`^{/pattern}` peel operator support** (`grit-lib/src/rev_parse.rs`)
   - In `apply_peel()`, added branch for peel payloads beginning with `/`.
   - Implemented `resolve_commit_message_search_from(repo, start, pattern)`:
     - BFS over commits reachable from the specified starting object.
     - Returns first commit whose message matches the search pattern.

3. **Regex-like matching for commit message search** (`grit-lib/src/rev_parse.rs`)
   - Introduced `pattern_matches_message(pattern, message)` helper used by:
     - global `:/pattern` search
     - object-local `^{/pattern}` search
   - Supports:
     - direct substring
     - simple wildcard conversion (`*` → `.*`, `?` → `.`)
     - anchored with `^...$` for exact regex compatibility in this scope
   - This makes `b.*` match commit message `brackets` for test expectations.

4. **More permissive approxidate for reflog selectors** (`grit-lib/src/rev_parse.rs`)
   - `approxidate()` now recognizes `now` prefix (case-insensitive), including strings like:
     - `now`
     - `now [or thereabouts]`
   - Returns current unix timestamp for `now*`.
