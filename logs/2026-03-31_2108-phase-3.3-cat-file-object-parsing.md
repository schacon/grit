## Phase 3.3 - cat-file object parsing

- Updated `gust cat-file -p` tree rendering to derive each entry's displayed type from the referenced object's actual kind (`blob`, `tree`, `commit`, or `tag`) instead of inferring only from mode.
- Added a safe fallback for unreadable tree-entry targets so pretty-print still emits Git-like type labels by mode (`tree` for `040000`, `commit` for `160000`, otherwise `blob`).
- Kept `blob` and `commit` pretty-print behavior aligned with Git's `-p` path (raw blob stream and raw commit payload output).
