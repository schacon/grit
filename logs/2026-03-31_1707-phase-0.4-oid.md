## Phase 0.4 — Object ID and hashing

- Updated `gust-lib/src/objects.rs` to make `ObjectId` a documented, public newtype API for SHA-1 object IDs.
- Added explicit hex parsing/encoding helpers:
  - `ObjectId::from_hex`
  - `ObjectId::to_hex`
  - `Display` and `FromStr` round-trip through 40-hex form.
- Added loose-object path helpers for canonical layout:
  - `loose_prefix` (`ab`)
  - `loose_suffix` (`cdef...`)
  - `loose_relative_path` (`objects/ab/cdef...`)
  - `loose_path_in` (`<objects-dir>/ab/cdef...`)
- Added conversion helpers and traits:
  - `from_array`
  - `AsRef<[u8]>`
  - `From<[u8; 20]>`
  - `TryFrom<&[u8]>`
- Updated `gust-lib/src/odb.rs` to use `ObjectId::loose_path_in` for object path construction.
- Updated `plan.md` to mark **0.4 Object ID and hashing** as completed (`[x]`).
- Added tests covering OID hex round-trip and loose object path layout.
