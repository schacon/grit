# Phase 10.1 — Loose refs and HEAD consistency

## Scope

- Implemented only phase 10.1 work for loose ref create/update/delete behavior.
- Focused changes in:
  - `grit-lib/src/refs.rs`
  - `grit/src/commands/update_ref.rs`
  - `plan.md` (10.1 checkbox only)

## What changed

- Added `resolve_refname_for_update(git_dir, refname, no_deref)` in `refs.rs`.
  - Follows symbolic refs (including `HEAD`) to the final loose target when `no_deref` is false.
  - Keeps the original ref name when `no_deref` is true.
  - Handles detached `HEAD` by leaving target as `HEAD`.
  - Limits symbolic depth to avoid cycles.
- Updated `grit update-ref` to mutate the resolved storage refname for:
  - single update
  - single delete
  - stdin batch `update`, `create`, and `delete`
- Reflog appends now use the resolved storage refname so logs track the actual updated loose ref.

## Tests

- Added unit tests in `refs.rs`:
  - symbolic `HEAD` dereferences to branch target by default
  - `--no-deref` keeps `HEAD` as mutation target
  - detached `HEAD` remains `HEAD` as mutation target
