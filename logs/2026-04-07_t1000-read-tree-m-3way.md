# t1000-read-tree-m-3way

## Summary

Implemented Git-compatible index validation and worktree up-to-date checks for `read-tree -m` three-way merges, removed incorrect preservation of stage-0 paths outside the three trees, and added symmetric 4-tree merge (`T0 T1 T1 T0`) for case #16.

## Code

- `grit/src/commands/read_tree.rs`
  - `validate_three_way_index_stage0` — matrix from `t1000-read-tree-m-3way.sh` (paths must match O/A/B rules; case 14 allows index matching A or B).
  - `require_uptodate` — uses `worktree_matches_entry` where Git requires “up-to-date”.
  - `(Some, Some, None)` — always require index blob to match ours and worktree to match index (cases 9–10).
  - `four_tree_symmetric_merge` — conflict stages 2/3 when `T0` and `T1` disagree; allow up to 4 tree arguments when `T0==T3` and `T1==T2`.
  - Max trees: 3 → 4 for merge.
