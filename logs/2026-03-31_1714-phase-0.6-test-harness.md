# Phase 0.6 - Test harness

## Goal

Implement a maintainable process to copy selected upstream scripts from `git/t/`
into `./tests`, substitute `git` command tokens with `gust` where appropriate,
and run the selected scripts.

## What was added

- `tests/harness/selected-tests.txt`
  - Initial selected upstream script set:
    - `t0001-init.sh`
- `tests/harness/t0001-init-allowlist.txt`
  - Initial allowlisted upstream test cases (small known-green subset):
    - `init notices EEXIST (1)`
    - `init notices EEXIST (2)`
- `tests/harness/refresh_from_upstream.py`
  - Reads upstream `git/t/t0001-init.sh`.
  - Extracts allowlisted `test_expect_success` blocks.
  - Rewrites command token `git` to `gust` using a boundary-safe regex that does
    not alter `.git`, `GIT_*`, or path fragments.
  - Regenerates `tests/t0001-init.sh`.
- `tests/harness/refresh.sh`
  - Wrapper to regenerate selected scripts.
- `tests/harness/run.sh`
  - Ensures `GUST_BIN` is available (builds `target/debug/gust` if needed).
  - Runs selected scripts from `tests/harness/selected-tests.txt`.

## Refresh process

From repo root:

```sh
tests/harness/refresh.sh
```

To expand coverage, add new descriptions to
`tests/harness/t0001-init-allowlist.txt`, or extend
`refresh_from_upstream.py` for more scripts listed in
`tests/harness/selected-tests.txt`.
