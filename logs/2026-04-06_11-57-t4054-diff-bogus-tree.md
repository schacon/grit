### Root-cause notes
- `diff-tree` accepted `-R` as a no-op; output rendering never swapped old/new sides. This produced incorrect raw reverse output for the null-sha1 case.
- Patch output paths (`diff-tree -p`, `diff-index -p`) produced diffs even when one side referenced a bogus all-zero blob OID for a non-add/delete entry; upstream instead fails with `error: bogus object <zero-oid>`.

### Fixes implemented
- Added `reverse: bool` option plumbing in `diff-tree`:
  - parser now toggles `reverse` for `-R`;
  - two-tree, one-commit, and `--stdin` commit/tree paths now call a shared `apply_reverse` helper before filtering/printing;
  - reverse handling swaps old/new path/mode/oid, status add/delete, and rename/copy direction.
- Added bogus-zero-object guard in patch writers:
  - `diff-tree` patch path now rejects entries where either side is zero OID for non-add/delete status with:
    - `error: bogus object 0000000000000000000000000000000000000000`
  - `diff-index` patch path mirrors the same guard, aligning with test expectations.
