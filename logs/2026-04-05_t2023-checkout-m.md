## t2023-checkout-m — 2026-04-05

- Claimed next near-complete target after `t2104`: `t2023-checkout-m` (2/5).
- Reproduced failures with:
  - `./scripts/run-tests.sh t2023-checkout-m.sh` → 2/5
  - `GUST_BIN=/workspace/tests/grit TEST_VERBOSE=1 bash tests/t2023-checkout-m.sh`
- Root causes:
  1. `checkout -m -- <path>` path mode did not restore conflicted content after a file had been resolved and re-added to stage 0.
  2. `checkout -m <branch>` branch switch mode behaved like plain checkout and rejected local changes instead of performing a 3-way merge.
  3. For `checkout -m <branch>`, conflict stage ordering for index entries was opposite expected semantics for tests (`:2` should be target branch, `:3` should local branch/working tree side).

- Implementation in `grit/src/commands/checkout.rs`:
  - Added branch-mode `--merge` handling:
    - New `switch_branch_with_merge()` path in command dispatch.
    - Computes merge base between current HEAD and target branch.
    - Builds stage 1/2/3 entries for conflicts and writes conflict-marker content to worktree.
    - Carries clean-merged entries as stage 0.
  - Added path-mode `checkout -m -- <path>` handling:
    - New `restore_conflicted_paths_from_merge_head()` helper.
    - Uses `HEAD` + `MERGE_HEAD` + merge base to recreate original conflict markers for listed paths.
    - Supports both exact and directory/glob pathspec matching logic already present.
  - Added local helper `stage_entry_local()` for setting stage bits on copied index entries.
  - Fixed stage ordering to match expected test semantics:
    - stage 2 = target branch side (`main` in `checkout -m main`)
    - stage 3 = local branch/worktree side (`topic`)
