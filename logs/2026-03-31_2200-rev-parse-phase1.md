# Rev-parse phase 1 (items 1.1, 1.2, 1.4 subset)

## Scope taken

- Implemented `gust rev-parse` for plan items:
  - `1.1` repository/non-repository discovery flags needed by selected tests.
  - `1.2` basic revision/object parsing and `--verify` behavior in scope.
  - `1.4` selected script ports (`t1500`, `t1503`) without parseopt-heavy coverage.

## Code changes

- Added `gust-lib/src/rev_parse.rs` with library helpers for:
  - optional repository discovery (`discover_optional`);
  - working tree / git-dir location checks and prefix handling;
  - revision parsing for full OID, abbreviated OID, and DWIM refs;
  - peel operators `^{}` / `^{object}` / `^{commit}`;
  - unique abbreviation generation and relative path formatting.
- Exported module via `gust-lib/src/lib.rs`.
- Replaced `gust/src/commands/rev_parse.rs` stub with working implementation:
  - discovery flags: `--is-inside-work-tree`, `--is-inside-git-dir`, `--is-bare-repository`, `--show-toplevel`, `--show-prefix`, `--git-dir`;
  - verification modes: `--verify`, `-q/--quiet`, `--default`, `--short[=<n>]`, `--end-of-options`;
  - baseline non-verify revision printing for valid revisions.
- Updated `gust/src/main.rs` to make top-level `--git-dir` non-global, so `gust rev-parse --git-dir` is parsed as a subcommand option.

## Test ports

- Added `tests/t1500-rev-parse.sh` (discovery flags subset).
- Added `tests/t1503-rev-parse-verify.sh` (`--verify` behavior subset including peeling and short hashes).
- Added both scripts to `tests/harness/selected-tests.txt`.

## Validation

- `cargo fmt` passed.
- `cargo clippy --workspace --all-targets -- -D warnings` passed.
- `cargo test --workspace` passed.
- `tests/t1500-rev-parse.sh` passed (`8/8`).
- `tests/t1503-rev-parse-verify.sh` passed (`8/8`).

## Remaining known gaps in rev-parse

- Pathspec quoting/magic path handling (`1.3`) still pending.
- Parseopt and broader rev-parse mode coverage intentionally deferred for this increment.
