## Phase 1.1 - init layout

Implemented Git-style repository skeleton details for `init_repository`:

- Ensured default init layout includes `objects`, `objects/info`, `objects/pack`,
  `refs`, `refs/heads`, `refs/tags`, `info`, and `hooks`.
- Added built-in default template installation (when no `--template` directory is passed):
  - `info/exclude`
  - `hooks/*.sample` files that match Git's default hook sample names
  - executable bit set on hook samples on Unix
- Kept default `HEAD` symbolic target as `ref: refs/heads/<initial_branch>`.
- Removed stray debug `eprintln!` output from `gust init`.

Validation steps to run:

- `cargo fmt`
- `cargo clippy --workspace --all-targets -- -D warnings`
- `cargo test --workspace`
