## Task: t4064-diff-oidfind

### Claim
- Claimed from `PLAN.md` after completing `t4055-diff-context`.
- Baseline in local harness: `5/10` passing.

### Baseline failures
- `find the greeting blob` failed because revision peeling rejected `^{blob}`:
  - `invalid ref: unsupported peel operator '{blob}'`.
- `find a tree` failed because `git log -t` was rejected by argument parsing.
- Merge-related cases failed early during `git merge --no-commit`:
  - merge attempted to read gitlink OIDs as blobs during worktree checkout and errored with `object not found`.
- Combined merge checks using:
  - `git diff-tree --format=%s --find-object=$blob -c --name-status HEAD`
  lacked command support for `--find-object`, `-c` combined status output, and `--format=%s` subject formatting.

### Root causes
1. `grit-lib rev_parse` supported `^{}`, `^{object}`, `^{commit}`, and `^{tree}`, but not `^{blob}`.
2. `grit log` did not accept compatibility `-t`.
3. `merge` worktree checkout tried to read every stage-0 index OID as a local blob, including gitlinks (`160000`) that point to submodule commits outside the repository object store.
4. `diff-tree` lacked find-object filtering and combined merge status handling for `-c` + `--name-status`, and ignored `--format=%s`.
5. `log --find-object` object matching only considered diff entry OIDs and missed tree-object presence transitions.

### Implemented fixes
- `grit-lib/src/rev_parse.rs`
  - Added `^{blob}` peel support in `apply_peel`, requiring peeled object kind `Blob`.

- `grit/src/commands/log.rs`
  - Added hidden compatibility parser flag for `-t`.
  - Extended `commit_has_object` with recursive tree containment checks so `--find-object=<tree-ish:path>` tree targets are detected via old/new tree presence transitions.

- `grit/src/commands/merge.rs`
  - In `checkout_entries`, skip gitlink (`mode 160000`) entries before blob reads to avoid submodule-commit lookup failures during merge checkout.

- `grit/src/commands/diff_tree.rs`
  - Added parsing for:
    - `--find-object=<spec>` / `--find-object <spec>`
    - `-c` / `--cc` (combined diff mode)
    - `--format=<template>` (wired into commit-header formatting path)
  - Added find-object resolution/filtering against diff entries.
  - Implemented combined `--name-status` rows for merge commits (`-c`) by composing per-parent status letters.
  - Implemented `--format=%s` commit header rendering (subject + blank separator) for diff-tree output.
