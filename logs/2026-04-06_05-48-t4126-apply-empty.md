### Baseline failures
- Failing assertions: 3, 4, 6, 7, 8.
- Observed runtime errors:
  - `git apply --allow-empty ...` rejected as unknown option.

  - setup reverse patch generation failed at `git diff -R ...` with `unknown revision: '-R'`.
  - funny-path sequence in test 8 failed as a consequence.

### Implemented fixes
1. **`git apply --allow-empty` support**
   - Added `--allow-empty` flag to `apply` args.
   - Added empty-input guard: patch streams with no parsed file patches now fail by default and only succeed with `--allow-empty`.

2. **Zero-preimage hunks can create missing targets**
   - Added `can_apply_with_empty_preimage()` helper for hunks with old side `0,0` and only added lines.
   - Updated worktree precheck to allow absent source path when patch shape indicates empty preimage.
   - Updated worktree/index apply paths to treat missing source as empty content for that patch shape.
   - Updated `--check` path to use the same empty-preimage behavior.

3. **`--check --apply` semantics**
   - Updated `apply` control flow so `--check --apply` performs check then continues into apply mode (instead of returning after check-only).

4. **`git diff -R` support needed by test setup**
   - Added `-R/--reverse` argument to `diff`.
   - Added trailing-arg flag recovery for `-R`.
   - Added entry reversal logic (swap old/new paths, modes, oids, and Added↔Deleted status).
