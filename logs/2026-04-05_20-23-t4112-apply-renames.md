## Task: t4112-apply-renames

### Claim
- Claimed after `t4044-diff-index-unique-abbrev` was moved back to blocked/pending.
- Marked as `[~]` in `PLAN.md`.

### Baseline
- `./scripts/run-tests.sh t4112-apply-renames.sh` reported `0/2` passing.
- Direct run showed both failures were path handling failures:
  - `error: failed to read include/arch/cris/klibc/archsetjmp.h: No such file or directory`
  - `error: failed to write include/arch/cris/klibc/archsetjmp.h: No such file or directory`

### Root cause
- `grit apply` always used one “effective path” for both preimage reads and postimage writes.
- Rename/copy patches in `t4112` require reading preimage from `old_path` and writing postimage to `new_path`.
- Multi-file patches in the same input also rely on using an original source snapshot for later copy hunks after earlier rename hunks mutate/remove the source file.

### Fix implemented
- Updated `grit/src/commands/apply.rs`:
  - Added `FilePatch::source_path()` and `FilePatch::target_path()` helpers.
  - Updated `--check`, worktree apply, index apply, and `--index` preflight to use source-vs-target path semantics.
  - Added source-content snapshotting for cross-path rename/copy hunks so subsequent sections read the original preimage content.
  - In `apply_to_index`, introduced an `original_index` snapshot and used it for cross-path source lookups to avoid order-dependent failures.
  - For rename application, remove source path from worktree/index after successfully writing the target path.
