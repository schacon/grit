## t0014-alias — 2026-04-07

### Goal
Make `tests/t0014-alias.sh` fully pass by aligning alias lookup/expansion behavior, loop handling, shell tracing, and help output with upstream Git expectations.

### Baseline
- `./scripts/run-tests.sh t0014-alias.sh` → **4/21**
- Key failures:
  - no alias-loop diagnostics for deprecated builtin alias cycles,
  - no `run_command` trace line for dashed external fallback,
  - shell alias `start_command` trace formatting mismatch,
  - missing subsection alias syntax support (`alias.<name>.command`, UTF-8/case-sensitive),
  - valueless alias keys not reported as config errors,
  - `help -a` did not list configured aliases.

### Implementation

#### 1) Alias resolution and subsection syntax
- Updated alias loading in `grit/src/main.rs`:
  - `get_alias_definition()` now returns `Result<Option<String>>` to surface valueless-key errors.
  - Added specificity-based matching for:
    - `alias.<name>`
    - `alias..name`
    - `alias.<name>.command`
  - This enables subsection syntax and preserves case/UTF-8 alias names from config.

#### 2) Alias shadowing for deprecated builtins
- Added `DEPRECATED_ALIAS_SHADOWABLE` (`whatchanged`, `pack-redundant`).
- Alias expansion now runs for unknown commands **and** for these deprecated builtins, matching `git --list-cmds=deprecated` + alias-shadow tests.
- `--list-cmds=deprecated` now returns the deprecated set so test scripts can iterate that category.

#### 3) Alias loop detection diagnostics
- Added recursive alias stack tracking in `run_alias_inner(...)`.
- On loop detection, emit Git-style diagnostics:
  - `'<from>' is aliased to '<to>'`
  - `fatal: alias loop detected: expansion of '<start>' does not terminate:`
  - loop chain with `<==` / `==>` markers.

#### 4) Shell alias trace compatibility
- Added shared trace helpers for `GIT_TRACE`.
- For shell aliases (`!cmd`), emit `trace: start_command: ...` in expected format.
- Command execution switched to `sh -c '<cmd> "$@"' '<cmd>' <args...>` to preserve expected `$*` argument behavior and trace payload.

#### 5) External dashed fallback and tracing
- Added `try_run_dashed_external_command(...)` in main dispatch fallback:
  - attempts `git-<subcmd>` when subcommand is unknown,
  - emits `trace: run_command: git-<subcmd> ...` before spawn,
  - propagates subprocess exit status,
  - falls back to normal unknown-command suggestion flow on ENOENT.

#### 6) Help output alias listing
- Updated `grit/src/commands/help.rs`:
  - load config aliases from `alias.*` and `alias.*.command`,
  - include aliases in `help -a` output.

#### 7) Unknown command wording alignment for alias tests
- Adjusted unknown-command messages to `not a git command. See 'git --help'.` so alias test expectations match upstream phrasing in this file.
