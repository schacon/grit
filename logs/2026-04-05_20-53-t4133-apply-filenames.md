  - failing tests:
    - `apply diff with inconsistent filenames in headers`
    - `apply diff with new filename missing from headers`
    - `apply diff with old filename missing from headers`

### Root cause
- `grit apply` accepted malformed patch headers and only failed later while reading files,
  emitting generic I/O errors instead of Git-compatible validation errors.
- Missing validations:
  - `diff --git` path must match `---`/`+++` header filenames.
  - For new/delete transitions, both sides of filename information must be present.

### Fix implemented
- Updated `grit/src/commands/apply.rs` in `run()` with early header validation before apply/check:
  - detect inconsistent old/new filename pairs and bail with:
    - `inconsistent old filename`
    - `inconsistent new filename`
  - detect missing required filename information for add/delete style patches and bail with:
    - `patch lacks filename information`
