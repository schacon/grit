### Initial diagnosis
- Current merge flow does not honor merge attributes (`-merge`, `merge=union`, custom `merge=<driver>`).
- `git checkout -m <path>` behavior for conflicted path recreation is incomplete.
- `conflict-marker-size` attribute warning/handling is not implemented.
- Custom merge driver execution and failure semantics are missing from merge core.

### Implementation summary
- Extended attribute parsing in `grit-lib/src/crlf.rs`:
  - Added `merge` attribute support (`MergeAttr::{Unspecified,Unset,Driver}`).
  - Added `conflict-marker-size` attribute capture.
- Integrated merge-attribute behavior in `grit/src/commands/merge.rs`:
  - Per-path merge behavior resolver now honors:
    - `-merge` (`merge` unset) as binary/non-text merge behavior.
    - `merge=union` by mapping unresolved conflicts to union output.
    - `merge=<driver>` by executing configured `merge.<name>.driver`.
  - Implemented custom merge driver command execution with placeholder expansion:
    - `%O %A %B %P %S %X %Y`.
  - Signaled custom-driver termination now maps to Git-compatible:
    - stderr: `error: failed to execute internal merge`
    - exit code: `2`.
  - Conflict marker size now respects `conflict-marker-size`; invalid values emit:
    - `warning: invalid marker-size '<value>', expecting an integer`.
  - Binary conflict messaging now emits warning-prefixed line expected by attr suite while preserving existing binary conflict text.
- Added merge-specific error mapping in `grit/src/main.rs`:
  - Detect internal merge execution failures and exit with code `2`.
- Implemented `checkout -m <path>` conflicted-path re-materialization in `grit/src/commands/checkout.rs`:
  - Reconstructs conflict markers from index stages 1/2/3.
  - Applies marker size from attributes and emits invalid marker-size warning.
  - Keeps non-merge checkout semantics unchanged.
