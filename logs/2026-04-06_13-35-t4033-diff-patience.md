### Implementation
- Added explicit diff algorithm selection wiring for patch generation:
  - `grit-lib::diff` now exposes `unified_diff_with_prefix_and_funcname_and_algorithm(...)`.
  - `grit diff` computes effective algorithm precedence (CLI > attributes driver algorithm > config > default).
  - `--patience`, `--histogram`, `--minimal`, and `--diff-algorithm[=<name>]` recovered from trailing args now correctly affect diff generation.
- Fixed attr-source behavior for bare repositories:
  - `resolve_attr_source_tree` now peels commit-ish values to tree OIDs (instead of treating commit OIDs as tree objects).
  - `diff --attr-source=<tree-ish>` now loads `.gitattributes` from the selected tree and applies driver algorithm configuration.
  - Invalid attr source values fail with non-zero status (`bad --attr-source or GIT_ATTR_SOURCE`).
- Improved `--no-index` behavior:
  - `diff --no-index --stat` now emits Git-style bars (`++++++++++-----------`) instead of only totals.
  - `diff --no-index` now applies resolved algorithm choice to unified patch generation.
- Fixed patch re-application flow used by `t4033`:
  - `apply` now uses old-side preimage path when patch headers differ old/new filenames without explicit rename headers, allowing no-index diff output to apply correctly after `mv file2 expect`.
