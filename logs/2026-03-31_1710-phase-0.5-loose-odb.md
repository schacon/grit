## Phase 0.5 — Loose object store

- Completed loose-object read path in `grit-lib/src/odb.rs`:
  - Read and zlib-decompress loose object files under `objects/<2>/<38>`.
  - Parse object headers in canonical form `"<type> <size>\0<data>"`.
  - Validate object kind is one of `blob`, `tree`, `commit`, or `tag`.
  - Validate payload size against the declared header size.
- Added typed error handling for object corruption classes:
  - `Error::WrongObjectType` for invalid header type.
  - `Error::TruncatedObject` for truncated streams/payloads.
  - `Error::CorruptObject` for malformed but non-truncated headers/content.
- Completed write path for canonical loose-object storage:
  - Serialize canonical store bytes.
  - SHA-1 object ID derivation from store bytes.
  - zlib-compress and atomically write to `objects/<2>/<38>`.
  - No overwrite when object already exists.
- Added/updated unit coverage for loose-object parsing edge cases:
  - Unknown type is rejected with `WrongObjectType`.
  - Missing header terminator / short payload is treated as truncation.
