## Phase 9.1 — commit-tree object construction

- Scope: implement only `plan.md` item `9.1`.
- Added tree/parent object kind validation in `grit commit-tree`:
  - tree argument must resolve to a `tree` object.
  - each `-p` parent must resolve to a `commit` object.
- Kept timestamp sourcing at CLI boundary (`current_unix_timestamp()` in command code); no library API now-calls added.
- Implemented injected-date parsing for identity headers:
  - supports raw epoch forms (`@<secs> <tz>`, `<secs> <tz>`, and `<secs>`).
  - supports `YYYY-MM-DD HH:MM` and `YYYY-MM-DD HH:MM:SS`.
  - honors `TZ=GMT`/`TZ=UTC` for human-readable dates used by `t1100`.
  - falls back to injected default offset when no explicit timezone is supplied.
- Preserved existing commit message input behavior from `-m`, `-F`, and stdin.
