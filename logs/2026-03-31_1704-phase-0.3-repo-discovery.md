### Implemented

- Tightened `.git` path validation during upward discovery:
  - Accept `.git` directory.
  - Accept `.git` regular file gitfile indirection.
  - Accept symlink to directory and symlink to regular file.
  - Reject unsupported `.git` filesystem node types with explicit error category.
- Added strict gitfile parsing:
  - Require `gitdir:` prefix line.
  - Return invalid-gitfile-format class error when malformed.
- Implemented `GIT_DIR` handling with path resolution from cwd.
- Implemented minimal `core.bare` parsing from `$GIT_DIR/config` (`[core] bare = ...`) for bare/non-bare detection when no explicit work tree is provided.
- Implemented work tree resolution precedence:
  1. `GIT_WORK_TREE` env var
  2. explicit work tree argument from caller
  3. `core.bare` defaulting behavior
- Added helper utilities in `repo.rs`:
  - git-dir shape check
  - relative path absolutization
  - minimal Git boolean parsing
- Added new typed library errors in `gust-lib/src/error.rs`:
  - `InvalidGitfileFormat`
  - `InvalidGitPath`
