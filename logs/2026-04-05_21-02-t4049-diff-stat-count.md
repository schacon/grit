### Findings so far
- `grit diff --stat --stat-count=<n>` undercounts total files in summary when:
  - mode-only changes are present,
  - unmerged stages should contribute to file count semantics expected by the test.
- Output truncation row (`...`) and "N files changed" summary counting logic appears to be based on rendered rows rather than true affected-file totals.
- Local harness parity issue identified:
  - `tests/t4049-diff-stat-count.sh` references `tests/test-binary-1.png`, but this fixture is missing in the local `tests/` mirror.
  - Added fixture locally by copying `git/t/test-binary-1.png` to `tests/test-binary-1.png`.

### Fixes implemented (current cycle)
- `grit diff --stat`/`--shortstat` counting:
  - Count `N files changed` by unique paths, not by rendered rows.
  - For binary entries, do not contribute insertion/deletion counts.
  - For unmerged entries, do not contribute insertion/deletion counts.
  - Only append insertion/deletion summary terms when at least one changed file exists.
- `grit diff --stat` rendering:
  - Added `Unmerged` marker rows to stat output path.
  - Count-column width now derives from rendered rows under `--stat-count`, reducing over-wide numeric padding.
- `grit reset --hard` working tree mode restore:
  - During checkout, non-executable blobs are now explicitly set to `0644` (not only executable entries set to `0755`).
  - This prevents executable mode leakage across test cases after hard reset.

- Reproduced remaining failure set after recent diff whitespace/max-depth work:
  - local mirror: `2/4` passing (`mode-only` and `unmerged` summary-count tests failing).
- Root cause for test 4:
  - `grit diff` uses `grit_lib::diff::diff_index_to_worktree`, which skipped all non-stage-0 index entries and therefore omitted conflict paths entirely.
  - Git behavior for conflicted paths in this scenario includes:
    - an `Unmerged` stat row, and
    - a second modified-content row for worktree-vs-conflict-stage content.
- Implemented in `grit-lib/src/diff.rs`:
  - collect non-stage-0 entries by path and select a comparison base stage with preference `2 > 3 > 1`;
  - emit `DiffStatus::Unmerged` entries for conflict paths;
  - when a worktree file exists and differs from the selected stage entry, emit an additional `DiffStatus::Modified` entry using worktree mode/OID.
