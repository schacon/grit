## Task

Make `t1004-read-tree-m-u-wf.sh` fully pass by implementing missing `read-tree -m -u` working-tree safety checks and compatibility behavior.

## Investigation

- Ran `GUST_BIN=/workspace/target/release/grit TEST_VERBOSE=1 bash t1004-read-tree-m-u-wf.sh`.
- Initial failures were concentrated in:
  - untracked-file clobber protection for two-way and three-way read-tree merges,
  - local-modification protection while updating tracked paths,
  - unsupported `--exclude-per-directory=.gitignore` behavior expected by test helper wrappers,
  - missing legacy strategy command aliases `merge-resolve` and `merge-recursive`.
- Verified test script semantics from:
  - `tests/t1004-read-tree-m-u-wf.sh`,
  - `tests/lib-read-tree.sh`,
  - upstream copies in `git/t/`.
- Read `git-read-tree` docs for expected merge/update safety model.

## Implementation

### 1) `read-tree` argument compatibility

File: `grit/src/commands/read_tree.rs`

- Added parser support for:
  - `--exclude-per-directory <value>`
- Implemented compatibility constraints:
  - only `.gitignore` is accepted,
  - option requires `-u`,
  - any other value errors with a clear message.

### 2) Pre-update worktree safety validation

File: `grit/src/commands/read_tree.rs`

- Added `validate_worktree_updates(...)` and integrated it before checkout when `-u` is enabled.
- Validation now:
  - detects untracked files that would be overwritten by added/updated merge outputs,
  - detects local tracked-file edits that would be overwritten by read-tree update,
  - allows overwrite of ignored untracked paths only when `--exclude-per-directory=.gitignore` is explicitly passed (via `IgnoreMatcher`).
- Added helper `worktree_matches_entry(...)` for tracked-file match checks against stage-0 pre-merge index entries.

### 3) Avoid unnecessary rewrites of unchanged tracked files

File: `grit/src/commands/read_tree.rs`

- `checkout_index_entries(...)` now:
  - computes stage-0 path sets only,
  - skips writing stage-0 entries when old and new blobs/modes are identical.
- This preserves local unrelated edits during `read-tree -m -u` updates as expected by t1004’s “our side” case.

### 4) Legacy merge strategy aliases

File: `grit/src/main.rs`

- Added dispatch support for:
  - `merge-resolve`
  - `merge-recursive`
- Implemented `run_legacy_merge_strategy(...)`:
  - accepts legacy form `<base> -- <ours> <theirs>`,
  - executes equivalent read-tree 3-way merge/update (`-m -u`) using command internals.
- This unblocks t1004’s final `D/F resolve` and `D/F recursive` expectations.
