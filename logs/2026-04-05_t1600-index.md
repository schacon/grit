# t1600-index

- Claimed and verified the `t1600-index` task from `plan.md`.
- Reproduced the initial upstream state at 5/7 passing, with failures in `index.skipHash config option` and `index version config precedence`.

## Root causes (detailed)

1. Harness helper gap: `test_trailing_hash` function missing from simplified `tests/test-lib.sh` caused case 6 to fail before exercising product behavior (addressed via harness helper or `tests/test_trailing_hash` where used).
2. Index serialization mismatch: v4 writing was downgraded to v2 in `grit-lib/src/index.rs`, causing config precedence checks to observe `2` instead of expected `4`.
3. `index.skipHash` compatibility missing in index writer and parser: trailing hash was always validated/written as SHA-1, not allowing all-zero trailer behavior expected by upstream test.

## Changes

- Updated `grit-lib/src/index.rs` to:
  - default invalid index-version fallbacks to version 2,
  - accept zeroed trailing index hashes,
  - honor `index.skipHash` and `feature.manyFiles` when writing indexes,
  - write on-disk index version 4 with pathname compression (shared-prefix compression + varint strip length; `shared_prefix_len()` / `write_varint()` helpers).
- Added `test-tool hexdump` support in `grit/src/main.rs` for upstream helper usage.
- Fixed the upstream runner's generated `t/helper/test-tool` wrapper in `scripts/run-upstream-tests.sh` to embed the active `grit` binary path where applicable.
