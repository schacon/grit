# t4112-apply-renames

- Claimed `t4112-apply-renames` from `plan.md` and inspected `git/t/t4112-apply-renames.sh`.
- Reproduced the failure with `CARGO_TARGET_DIR=/tmp/grit-build-t4112 bash scripts/run-upstream-tests.sh t4112-apply-renames 2>&1 | tail -40`; initial result was 0/2 passing.
- Traced the breakage to `grit/src/commands/apply.rs`: rename/copy patches were being treated like in-place edits of the destination path, so `--check` tried to read the destination before it existed and `--index` wrote the wrong source/destination paths.
- Split apply handling into explicit source and destination path helpers so rename/copy patches read hunks from the old path and write the new path.
- Updated worktree apply to create destination parents, keep copies, remove rename sources, and apply mode changes to the destination path.
- Updated index apply to read source blobs from the correct index entry, write destination entries, and remove renamed source entries.
- Found a second ordering bug: later `copy from` sections must use the original source preimage even if an earlier patch section already modified that source in the worktree.
- Fixed that by snapshotting rename/copy source file contents before the worktree apply loop mutates any files.
- Rebuilt with `CARGO_TARGET_DIR=/tmp/grit-build-t4112 cargo build --release -p grit-rs`.
- Verified targeted Rust tests with `cargo test -p grit-rs apply -- --nocapture` (2/2 passing).
- Verified upstream with `rm -rf /tmp/grit-upstream-workdir /tmp/grit-upstream-results && CARGO_TARGET_DIR=/tmp/grit-build-t4112 bash scripts/run-upstream-tests.sh t4112-apply-renames 2>&1 | tail -40` (2/2 passing).
- Ran `CARGO_TARGET_DIR=/tmp/grit-build-t4112 cargo fmt --all 2>/dev/null; true`.
- Marked `t4112-apply-renames` complete in `plan.md` and updated `progress.md` / `test-results.md`.
