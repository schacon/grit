# t7514-commit-patch

## Goal

Make `tests/t7514-commit-patch.sh` pass (hunk edit with `commit -p -m` and `commit --dry-run -p -m`).

## Changes

- Implemented interactive `commit -p` / `commit -i` in `grit/src/commands/commit.rs`:
  - Myers line diff index vs worktree, prompts matching Git (`Stage this hunk`, `y/n/q/a/d/s/e/?`).
  - Inverted blend flags vs stash: `y` stages worktree content (not index).
  - After `e`, EOF on the next prompt treats as `y` (matches Git / t7514 single-line stdin).
  - `--dry-run`: patch selection in memory only; no index write.
  - Pathspec: resolve targets for partial tree without pre-staging whole files like non-patch pathspec commit.
- Rejected `-a`, `-i/--include`, `-o/--only` with `-p` like Git.
- `stash::edit_bytes_tempfile`: prefer `GIT_EDITOR`, ignore `:` placeholders, fall back to `true` when non-tty (avoids vim hang; aligns with commit editor resolution).

## Validation

- `./scripts/run-tests.sh t7514-commit-patch.sh` → 3/3 pass
- `cargo test -p grit-lib --lib`
- `cargo fmt`, `cargo clippy --fix --allow-dirty -p grit-rs`
