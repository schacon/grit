# Phase 10.3 — update-ref batch stdin mode

## Scope

- Completed only `plan.md` item 10.3.
- Focused files:
  - `grit/src/commands/update_ref.rs`

## What changed

- Enforced `-z` usage with `--stdin` in `update-ref` argument handling.
- Reworked batch stdin parsing for both line-delimited (`--stdin`) and NUL-delimited
  (`-z --stdin`) modes, with command parsing for:
  - `update`, `create`, `delete`, `verify`
  - `option no-deref`
  - `start`, `prepare`, `commit`, `abort`
- Added quoted-argument parsing for line mode, including escaped characters and
  octal escapes used by upstream stdin command formats.
- Added transactional command-state handling so `start/prepare/commit/abort`
  produce status output (`<command>: ok`) and defer queued updates until commit.
- Split batch execution into operation helpers (`apply_update`, `apply_create`,
  `apply_delete`, `apply_verify`) to keep behavior consistent across direct and
  stdin-driven execution.
