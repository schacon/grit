# t3511-cherry-pick-x

## Goal

Make `tests/t3511-cherry-pick-x.sh` fully pass: cherry-pick `-x` / `-s` commit message formatting aligned with Git `sequencer.c` / `trailer.c`.

## Changes

- **`grit-lib`**
  - New `commit_trailers` module: trailer block scanning (inspired by Git `trailer.c`), `append_cherry_picked_from_line`, `append_signoff_trailer`, `finalize_cherry_pick_message` (stripspace when `-x` unless `commit.cleanup` is set; `strip` mode when `commit.cleanup=strip` and `-c` overrides).
  - `parse_commit`: set `raw_message` when body is not UTF-8 **or** when body does not end with `\n`, so `serialize_commit` does not inject an extra trailing newline.
- **`grit` `cherry_pick.rs`**
  - Use `finalize_cherry_pick_message` for the main pick path and empty-pick `MERGE_MSG`.
  - `cherry_pick_source_message`: when `raw_message` is set (no trailing LF), use lossy UTF-8 of raw bytes so the last line is not dropped vs `CommitData::message`.
  - `--continue`: append cherry + signoff with the same helpers; load config for trailer rules.
  - `committer_name_email` helper shared with committer line formatting.

## Tests

- `./scripts/run-tests.sh t3511-cherry-pick-x.sh` → 22/22
- `cargo test -p grit-lib --lib`

## Notes

- `last_line_looks_like_trailer` bridges the gap when Git’s trailer iterator still treats a final `Signed-off-by:` line as a trailer but our block scanner does not (messages without a final newline).
