## Implementation (complete)

### `grit am` option-surface + persistence (`grit/src/commands/am.rs`)
- Added `am` CLI support for:
  - `--whitespace=<action>`
  - `-C<n>` (minimum matching context)
  - `-p<n>` (strip components)
  - `--directory=<dir>` (path prefix)
- Extended persisted am session options to include:
  - `whitespace-fix`
  - `context=<n>`
  - `strip=<n>`
  - `directory=<dir>`
- Added legacy-compatible `rebase-apply/apply-opt` writer for active sessions so `--reject` sessions emit `--reject` in state (required by test 8 assertion).

### Reuse shared `apply` semantics from `am`
- Refactored `am` patch application path to delegate to shared `grit apply` command logic via in-memory patch input:
  - introduced `apply_patch_to_worktree(work_tree, diff, opts)` adapter that builds `apply::Args` from current am options and invokes apply engine directly.
- Removed custom ad-hoc patch application behavior in `am` that previously diverged from `git apply` semantics for strip/directory/context/whitespace/reject handling.
- Updated reject handling path to rely on shared apply behavior rather than bespoke `.rej` generation.

### `grit apply` reusable entrypoint (`grit/src/commands/apply.rs`)
- Added public helper:
  - `run_with_patch_input(args: Args, patch_input: Option<&str>) -> Result<()>`
- `run(args)` now delegates to `run_with_patch_input(args, None)`.
- Enables internal callers (like `am`) to apply patch text without stdin/files while preserving exact apply semantics.
