## Phase 6.2 - ls-tree pathspec / glob restriction semantics

- Implemented targeted pathspec restriction behavior in `gust/src/commands/ls_tree.rs` to match covered upstream semantics:
  - explicit pathspec descent into subtrees even without `-r` (e.g. `path2/`, `path2/baz`)
  - exact pathspecs for trees vs descendants (exact match does not imply child listing in non-recursive mode)
  - `-t` behavior for showing ancestor tree entries when descending toward a restricted path
  - literal treatment of pathspec text for covered wildcard-like cases (no glob expansion introduced)
- Added focused unit tests in `gust/src/commands/ls_tree.rs` for trailing-slash restriction, exact tree restriction, `-t` ancestor-tree visibility, descend detection, and cwd-relative display path conversion (`../...`) for outside-prefix cases.
