# t5571-pre-push-hook

## Summary

Made `tests/t5571-pre-push-hook.sh` pass 11/11 by aligning `grit push` with Git’s pre-push hook contract and refspec resolution.

## Changes (`grit/src/commands/push.rs`)

- **Pre-push stdin local ref column**: When refspec source is `HEAD`, keep `HEAD` in the hook line (not the resolved branch ref); use `rev_parse::resolve_revision` for non-ref sources like `HEAD~`.
- **`push.default=upstream`**: Default push uses `refs/heads/<branch>` → `branch.<branch>.merge` when that branch’s `remote` matches the push remote (same errors as Git when upstream missing or remote mismatched).
- **Hook stdin ordering**: When mixing `refs/...` local names with rev expressions, emit `refs/*` lines first (matches Git for `one:tag1 HEAD~:refs/heads/prev`).
- **Tag refspec short destination**: For `src:dst` with unqualified `dst`, if the resolved local ref is under `refs/tags/`, map to `refs/tags/<dst>` (not `refs/heads/<dst>`).
