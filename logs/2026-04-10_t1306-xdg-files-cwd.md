# t1306-xdg-files

## Issue

`t1306-xdg-files` failed test 10 (`$XDG_CONFIG_HOME overrides $HOME/.config/git/ignore`) even though Grit’s ignore logic matched Git when run from the nested repo directory.

## Root cause

The Gust TAP harness (`test-lib-tap.sh`) resets `cwd` to `$TRASH_DIRECTORY` before and after every `test_expect_success` body. Upstream Git’s `test-lib` does **not** do that, so after test 8’s `cd git`, tests 9–10 run inside `git/` and pathspecs like `to_be_excluded` resolve correctly.

With per-test `cd` to trash, `git add to_be_excluded` in the subshell for test 10 failed early (`pathspec did not match`), so the chain never reached `test_must_fail git add excluded_by_xdg_only`.

## Fix

Set `TEST_LIB_INHERIT_CWD=1` in `tests/t1306-xdg-files.sh` so `test_eval_inner_` skips forcing `$TRASH_DIRECTORY` around each eval, matching upstream cwd persistence for this file only.

## Verification

`./scripts/run-tests.sh t1306-xdg-files.sh` → 21/21.
