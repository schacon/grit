## t6010-merge-base

Date: 2026-04-05

### Goal
Make `tests/t6010-merge-base.sh` fully pass by implementing missing merge-base/show-branch options and related merge behavior.

### Reproduction
- Harness baseline:
  - `./scripts/run-tests.sh t6010-merge-base.sh` -> `6/12`.
- Direct run from `tests/`:
  - `EDITOR=: VISUAL=: LC_ALL=C LANG=C GUST_BIN=/workspace/target/release/grit bash t6010-merge-base.sh`
  - Failing tests: 3, 4, 8, 9, 10, 11.

### Root causes
1. `show-branch` only implemented listing mode; missing `--merge-base` and `--independent`.
2. `merge-base` missing `--fork-point`.
3. `merge` missing `--allow-unrelated-histories`.
4. `merge-base --fork-point` logic initially used reflog `new_oid` candidates for checkout entries, which incorrectly selected the tip of `derived` instead of the branch reset points from upstream reflog.

### Changes
- `grit/src/commands/show_branch.rs`
  - Switched to raw argument parsing mode.
  - Added `--merge-base` support using octopus merge-base semantics.
  - Added `--independent` support.
  - Preserved existing default list behavior.

- `grit/src/commands/merge_base.rs`
  - Added `--fork-point` mode and option parsing.
  - Implemented fork-point resolution using upstream reflog candidates plus ancestor filtering and best-candidate reduction.
  - Added fallback to normal merge-base when reflog-driven fork-point cannot be found.
  - Corrected checkout reflog handling: for `checkout:` entries, candidate uses `old_oid` (the branch being left), matching expected fork-point behavior in test loops.

- `grit/src/commands/merge.rs`
  - Added `--allow-unrelated-histories` flag to args.
  - Wired both single and octopus merge paths to allow empty-base merges when the flag is set.
  - Added helper to create an empty virtual base commit for unrelated-history merges.

- `grit/src/commands/pull.rs`
  - Updated merge args construction to include the new `allow_unrelated_histories` field.
