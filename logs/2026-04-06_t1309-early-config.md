- Target file: `tests/t1309-early-config.sh`

## Root causes
1. `test-tool config` subcommand did not exist in `grit`, so `test-tool config read_early_config ...` failed immediately.
2. Incompatible repository versions (`core.repositoryformatversion` too high) were not downgraded to warning+fallback behavior for early-config reads.
3. Invalid config file parse failures in repository-local config needed to be ignored for this helper path (matching upstream helper behavior for early read tests).

## Implemented fixes

### `grit/src/main.rs`
- Added `run_test_tool_config(rest: &[String])` and wired it into test-tool dispatch.
- Implemented support for:
  - `test-tool config read_early_config <key>`
- Behavior:
  - Discovers repository if present; otherwise uses global/system config only.
  - Loads config cascade and prints all values for the requested key in load order.
  - For incompatible repository format versions, prints warning:
    - `warning: Expected git repo version <= 1, found <version>`
    - then falls back to non-repo config scope for early-config read.
  - For local config parse errors in this helper path, falls back to non-repo config scope.

### `grit-lib/src/repo.rs`
- Updated repository format validation parser to return:
  - `Error::UnsupportedRepositoryFormatVersion(<n>)` for invalid or out-of-range `core.repositoryformatversion` values.
- This enables callers (like `test-tool config read_early_config`) to distinguish incompatible repo-version behavior from generic config errors and emit expected warning semantics.
