### Investigation Notes
- The first blocker (`unsupported option: -C`) was already removed by option parser updates.
- Remaining failures (tests 9 and 10) showed raw `D` output where `C100` was expected:
  - expected: `:100644 100644 <blob> <blob> C100	path1/COPYING	path0/COPYING`
  - actual:   `:100644 000000 <blob> 000000... D	path0/COPYING`
- Root cause: `diff-files` parsed `-C/--find-copies-harder/-R` but did not execute copy detection or reverse semantic mapping in output generation.

### Implemented Fix
- Updated `grit/src/commands/diff_files.rs`:
  - Added option plumbing fields in `Options`:
    - `reverse`
    - `find_copies`
    - `find_copies_harder`
  - Added conversion from internal `Change` to `grit_lib::diff::DiffEntry` via `changes_to_diff_entries(...)`.
  - Implemented reverse-side mapping (`-R`) during conversion.
  - Wired `grit_lib::diff::detect_copies(...)` for `-C` copy detection.
  - Added `collect_copy_sources(...)`:
    - normal mode uses index blobs
    - reverse mode uses worktree blobs
  - Switched raw output to `DiffEntry` rendering so `C###`/`R###` status and dual-path raw records are emitted.
