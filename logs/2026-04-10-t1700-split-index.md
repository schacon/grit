# t1700-split-index

- Implemented Git split index: `link` extension, `sharedindex.<sha1>`, EWAH delete/replace bitmaps, merge on read, write path matching `prepare_to_write_split_index` / `write_locked_index` behavior.
- `update-index`: `--split-index` / `--no-split-index`, sticky split when `split_link` present; `core.splitIndex` false disables implicit split; warnings match Git.
- `rev-parse --shared-index-path`; `test-tool dump-split-index`.
- `Repository::write_index_at_split` + `adjust_shared_perm` on main index; shared index temp permission adjustment.
- Alternate `GIT_INDEX_FILE` (not `$GIT_DIR/index`): unified index only, no `link` (t1700 #25).
- Dependency: `filetime` for freshening shared index mtime.
- `IndexEntry.base_index_pos`; `add_or_replace` preserves it across refresh.

Validation: `./scripts/run-tests.sh t1700-split-index.sh` (29/29), `cargo test -p grit-lib --lib`.
