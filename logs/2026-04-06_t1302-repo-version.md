## Failures addressed
1. `gitdir selection on unsupported repo` did not fail for format version `99`.
2. `gitdir required mode` (`apply --check --index`) did not fail on unsupported repos.
3. `rev-parse --git-dir` accepted unsupported extension combinations:
   - version 1 + `no-such-extension` (should fail)
   - version 0 + `noop-v1` (should fail)
4. `git prune` did not respect `extensions.preciousObjects=1`.
5. Harness prereq mismatch for `DEFAULT_REPO_FORMAT` caused wrong expectation in test 2.

## Implemented fixes

### `grit-lib/src/error.rs`
- Added repository-format specific errors:
  - `UnsupportedRepositoryFormatVersion(u32)`
  - `UnsupportedRepositoryExtension(String)`

### `grit-lib/src/repo.rs`
- Added repository format validation during `Repository::open()`:
  - supports core repository format versions `0` and `1`,
  - rejects versions `>1`,
  - for v1, rejects unknown extensions (with allowlist for known/used extensions),
  - for v0, permits unknown extensions except `*-v1` compatibility markers.
- Added helpers to resolve worktree `commondir` and locate shared config.

### `grit/src/commands/config.rs`
- For repository-scoped operations (default/local/worktree modes), validates the discovered repository before proceeding.
- This makes `git -C <unsupported-repo> config ...` fail as expected, while preserving file/global/system behavior.

### `grit/src/commands/apply.rs`
- Ensured `--check --index` path validates repository presence/format (required-mode behavior), so unsupported repos correctly fail.

### `grit/src/commands/prune.rs`
- Added `extensions.preciousObjects` guard:
  - `grit prune` now errors in precious-objects repositories (matching expected protective behavior).

### `scripts/run-tests.sh`
- Exported `_prereq_DEFAULT_REPO_FORMAT=set` in the test runner environment so repo-format prerequisite-dependent assertions use correct expectations in the simplified harness.
