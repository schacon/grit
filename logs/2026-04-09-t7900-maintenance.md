# t7900-maintenance

## Summary

- Replaced `grit maintenance` with a Git-oriented implementation: `run`, `is-needed`, `register`/`unregister`, `start`/`stop`, strategies, schedules, trace2 `child_start` for `test_subcommand`, `--detach` spawning a child, `GIT_TEST_MAINT_SCHEDULER` hooks for crontab/systemctl.
- Added `test_subcommand` / `test_subcommand_flex` to `tests/test-lib-harness.sh` (t7900 and others depend on them).
- `t7900-maintenance.sh`: override `test_expect_code` so bash does not leave `set -e` enabled after exit-code checks (avoids FATAL before `test_done`).
- `git gc`: hidden `--skip-foreground-tasks` for maintenance’s nested `gc` child; exported `need_to_gc` for auto conditions.
- `git commit`: calls `maintenance::run_auto_after_commit` after successful commit.
- `main.rs`: `trace2_emit_child_start_json` uses `Vec<String>`; `ExplicitExit` with empty message prints nothing (exit code preserved).
- Dependency: `atty` for default `--quiet` on maintenance run.
