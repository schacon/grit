## 2026-04-06 — t1411-reflog-show

### Goal
Make `tests/t1411-reflog-show.sh` fully pass by aligning `log -g` / `reflog show`
behavior with upstream Git for selector rendering, formatting, and reflog patch output.

### Initial failure snapshot
- `./scripts/run-tests.sh t1411-reflog-show.sh` → **7/17**
- Verbose failures showed:
  - Reflog header identity contained timestamp in `Reflog: ... (...)` (should be name+email only).
  - `git reflog --format=short` was parsed as top-level `reflog` arg parse error (missing implicit `show` subcommand).
  - Date selector rendering for `HEAD@{now}` / `--date=*` remained numeric index (`@{0}`) instead of date forms.
  - `log -g -p` failed in setup due to `update-ref ... HEAD^` not being accepted.

### Implementation notes

#### 1) Reflog CLI dispatch compatibility
- Added `preprocess_reflog_args()` in `grit/src/main.rs`.
  - Preserves `-<N>` shorthand conversion via existing log preprocessor.
  - Injects implicit `show` subcommand for reflog invocations that do not explicitly pass
    `show|expire|delete|exists|write`.
  - Fixes `git reflog --format=short -1` parsing compatibility.

#### 2) Reflog show wired through log -g engine
- Reworked `run_show()` in `grit/src/commands/reflog.rs` to delegate to
  `crate::commands::log::run(...)` with `walk_reflogs=true`.
- Mapped `--abbrev-commit` / `--no-abbrev-commit` to `log` abbreviation behavior using
  `abbrev` / `no_abbrev`.
- Defaulted `reflog show` to oneline-like output when no explicit format is provided,
  matching upstream reflog default output.

#### 3) Reflog selector/date rendering and header identity
- Extended reflog walk candidate model in `grit/src/commands/log.rs`:
  - Track selector style intent (`force index` vs `force date`) per input revision.
  - Track whether revisions include explicit selector syntax (`@{...}`).
  - Parse reflog entry timestamp and timezone offset from identity.
- Added selector helpers:
  - `parse_reflog_walk_revision()`
  - `format_reflog_selector()`
  - `format_reflog_selector_date()`
  - `parse_reflog_offset_from_identity()`
- Behavior now matches test expectations:
  - Explicit `@{0}` stays index-based, even with `--date=*`.
  - `@{now}` and date selectors render date-based selectors.
  - `--date=raw` and `--date=unix` affect `%gd` / selector output correctly.
  - `log.date` config does not implicitly alter reflog selector format.
- Reflog header identity now uses `format_ident_for_header(...)` (name+email only).

#### 4) Reflog patch output behavior (`log -g -p`)
- Added reflog-walk diff emission using commit-vs-parent semantics:
  - Convert parsed reflog commit into `CommitInfo` and call `write_commit_diff(...)`.
  - Ensures patch blocks are present and match non-reflog `git log -p` shape.
- Added blank-line spacing compatibility between reflog entries with patch output.

#### 5) Revision resolution compatibility for `HEAD^` in update-ref
- In `grit/src/commands/update_ref.rs`, expanded `resolve_oid_or_ref()`:
  - If direct OID / ref lookup fails, attempt full revision parser
    `grit_lib::rev_parse::resolve_revision(repo, s)`.
  - This allows inputs like `HEAD^` for update-ref commands used by t1411 setup.

#### 6) Branch reflog creation behavior
- In `grit/src/commands/branch.rs`, create a reflog entry on branch creation by default
  (not only when `--create-reflog` is passed), matching behavior expected by reflog walk
  tests and preserving previous t1414 expectations.
