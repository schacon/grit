## Task: t4116-apply-reverse

### Claim
- Claimed from `PLAN.md` after `t4126-apply-empty`.
- Baseline status before fixes:
  - local harness: `2/7`
  - upstream harness: `2/7`

### Baseline failures
- Reverse application cases involving binary patches failed:
  - payload decode/decompress mismatches on `GIT binary patch` literal sections,
  - reverse mode did not swap binary forward/reverse payload streams.
- Setup archive resolution also failed for revision-like names in this test flow.

### Implemented fixes
1. **Binary patch parsing in `git apply`**
   - Added parser support for `GIT binary patch` with `literal <size>` sections.
   - Implemented base85 payload decoding and zlib inflation into blob content.
   - Added binary patch payload storage on parsed file patches.

2. **Binary patch application paths**
   - Worktree apply path now writes inflated binary payloads (with mode handling).
   - Index apply path now writes inflated payloads as blob objects and updates index entries.
   - Added `--binary` alias for `--allow-binary-replacement`.

3. **Reverse mode correctness**
   - `-R` now swaps old/new OIDs as well as binary forward/reverse payloads.
   - Existing text hunk reversal and rename path behavior remain intact.

4. **Archive setup compatibility**
   - Tree-ish resolution in `archive` now accepts revision names used in test setup.
