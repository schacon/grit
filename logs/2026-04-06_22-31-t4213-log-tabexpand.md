### Root causes identified

1. `show` did not accept `--expand-tabs` / `--no-expand-tabs`, causing argument failures.
2. `show --pretty` without an explicit value did not behave like Git's default `medium`.
3. Pretty message rendering did not apply tab-expansion behavior expected by the tests.
4. `--pretty=short` required a distinct default expansion behavior (4-space) compared to other pretty formats.

### Implementation

- Updated `grit/src/commands/show.rs`:
  - Added CLI args:
    - `--expand-tabs[=<N>]` with default `8` when explicitly enabled.
    - `--no-expand-tabs`.
    - `--pretty` optional value defaulting to `medium`.
  - Added helper functions:
    - `resolve_expand_tabs(args)` to determine effective tab policy.
    - `format_pretty_message(message, expand_tabs, indent_lines)` for per-line rendering.
    - `expand_tabs_preserving_columns()` for fixed tab-stop expansion.
  - Applied pretty-message rendering across show formats:
    - default/medium/full/fuller/short/raw/email.
  - Matched Git-compatible defaults:
    - most formats expand tabs to width 8 by default,
    - `raw` and `email` preserve tabs,
    - `short` defaults to width 4.
