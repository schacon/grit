## Task: t4140-apply-ita

### Claim
- Claimed from `PLAN.md` after completing `t4116`.
- Baseline status before fixes:
  - local harness: `2/7`
  - upstream harness: `2/7`

### Baseline failures
- Failing assertions: 1, 2, 4, 6, 7.
- Observed runtime problems:
  - i-t-a setup produced non-git-compatible patch headers:
    - creation patch lacked `new file mode` and `/dev/null` header.
    - deletion patch used `index 0000000..0000000` instead of empty-blob preimage (`e69de29..0000000`).
  - `git apply --cached creation-patch` failed against i-t-a index entries by trying to read zero OID as a real blob (`object not found: 0000...`).
  - `git apply -N` support was missing and creation/complex apply did not preserve index intent-to-add entries.
  - `git apply --index creation-patch` incorrectly succeeded when target path was missing from worktree.

### Implemented fixes
1. **Persist i-t-a index metadata**
   - `grit add -N` now sets the index entry intent-to-add extended flag (`set_intent_to_add(true)`).
   - Index writer now promotes output version to v3 whenever any entry has extended flags, ensuring i-t-a metadata survives round-trip load/write.

2. **Git-compatible diff behavior for i-t-a entries**
   - Added `empty_blob_oid()` helper to `grit-lib` diff module.
   - `diff_index_to_worktree` now special-cases entries marked intent-to-add:
     - when file exists in worktree: emit `Added` against `/dev/null`.
     - when file is absent in worktree: emit `Deleted` from empty blob.
   - Diff header generation now uses empty-blob OIDs for added/deleted entries when a side carries zero OID, matching Git’s `index` line conventions for i-t-a patches.

3. **`git apply -N/--intent-to-add` support**
   - Added `-N/--intent-to-add` flag to `apply`.
   - For worktree apply mode (non-`--cached`, non-`--index`), patch-created files are now recorded in the index as intent-to-add entries (empty-blob OID + i-t-a flag).

4. **`--index` preflight parity for new-path patches**
   - `verify_worktree_matches_index` now validates `is_new` patch targets when an index entry already exists:
     - missing worktree path => `does not match index` error.
     - mismatched worktree/index blob => `does not match index` error.
   - This enforces Git-compatible failure for `git apply --index creation-patch` in the i-t-a missing-worktree scenario.
