## t4070-diff-pairs

### Baseline
- Claimed from plan as in-progress (`[~]`) at 1/7.
- `./scripts/run-tests.sh t4070-diff-pairs.sh` -> 1/7 initially.
- Direct test run showed two immediate blockers:
  - `git diff-pairs` was still passthrough/unsupported (`git: 'diff-pairs' is not a git command` path previously).
  - `git diff-tree -z ...` was unsupported in our implementation, preventing the stdin format expected by this test.

### Implementation summary

#### `grit/src/commands/diff_pairs.rs` (native implementation)
- Replaced passthrough behavior with a native parser/renderer:
  - parses `-z`, `--raw`, `-p/--patch`, compatibility `--abbrev*`, and rejects unsupported options.
  - enforces required `-z` input mode and rejects pathspec args with git-compatible `usage:` failures.
  - parses raw metadata records from stdin (`:oldmode newmode oldoid newoid status[score]`).
  - parses following NUL-terminated path payloads, including 2-path records for rename/copy.
  - rejects tree-object records with `fatal: tree objects not supported`.
  - supports explicit queue flush via empty token (double-NUL separator), preserving separator in output.
- Added raw output rendering that reproduces upstream raw NUL record layout byte-for-byte.
- Added patch output path by converting parsed pairs to `DiffEntry` and delegating to shared diff patch writer.

#### `grit/src/commands/diff.rs`
- Added `pub(crate) fn write_patch_from_pairs(...)` to expose a safe, stable patch-rendering entrypoint for `diff-pairs` without leaking private algorithm types.

#### `grit/src/commands/diff_tree.rs`
- Added `-z` parsing support for raw output mode.
- Implemented NUL-terminated raw emission (`write_raw_entry`) for `OutputFormat::Raw` and `--patch-with-raw` paths.
- Ensured rename/copy raw rows emit two NUL-separated paths and non-rename rows emit one path token.
- This unblocks `t4070` setup/expectation plumbing where `diff-tree -r ... -z` feeds `diff-pairs`.

#### `grit/src/commands/config.rs`
- Extended legacy clap parsing for `--unset` / `--unset-all` with `num_args = 0..=1` + `default_missing_value = ""`.
- Added legacy fallback behavior to read the key from positional args when the flag appears without an inline value.
- This preserves upstream-style cleanup calls used by test harness helper functions.
