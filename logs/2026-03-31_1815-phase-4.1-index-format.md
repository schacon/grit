## Phase 4.1 Index format parse/write

- Claimed and completed `plan.md` item `4.1`.
- Focused implementation in `grit-lib/src/index.rs` with minimal compatibility
  stubs for fixed index extensions.

### Implemented

- Extended `Index` with extension storage:
  - Added `extensions: Vec<IndexExtension>` preserving on-disk order.
  - Added `IndexExtension` enum with explicit variants for:
    - cache-tree (`TREE`)
    - split-index (`LINK`)
    - sparse (`SDIR`)
    - unknown signatures (opaque preservation)
- Kept index entry encoding/decoding in big-endian network byte order and added
  stronger truncation checks to avoid panics on malformed data.
- Updated parser flow:
  - Validates minimum file size for header + trailer checksum.
  - Parses declared entries first.
  - Parses trailing extensions as `signature + u32 size + payload`.
  - Preserves known and unknown extension payloads for round-trip writes.
- Updated serializer flow:
  - Writes header and entries.
  - Emits extensions back in original order with original payload bytes.
- Corrected `Default` semantics for `Index` by delegating to `Index::new()` so
  default version remains `2`.

### Tests

- Added extension-specific unit coverage in `grit-lib/src/index.rs`:
  - Round-trip preservation test for `TREE`/`LINK`/`SDIR`.
  - Structural error test for truncated extension payload.
