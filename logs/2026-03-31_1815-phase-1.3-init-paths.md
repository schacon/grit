## Phase 1.3 - init paths

Implemented path-sensitive init behavior for existing vs new repositories, aligned with Git semantics for the positional `directory` argument.

### Code changes

- Updated `gust/src/commands/init.rs`:
  - Resolved the default init target from `current_dir()` with an explicit error instead of a silent fallback.
  - Determined the effective git directory from:
    - `--separate-git-dir` when provided,
    - bare mode target path for `--bare`,
    - otherwise `<directory>/.git`.
  - Added reinitialization detection (`HEAD` + `objects/`) before init work.
  - Matched Git output messaging:
    - `Initialized empty Git repository in ...` for new repos.
    - `Reinitialized existing Git repository in ...` for existing repos.
