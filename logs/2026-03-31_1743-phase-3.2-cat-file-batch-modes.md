## Phase 3.2 - cat-file batch modes

- Implemented Git-style batch option forms for `cat-file`: `--batch[=<format>]`, `--batch-check[=<format>]`, and `--batch-command[=<format>]`.
- Added batch format placeholder support needed for the t1006 port path: `%(objectname)`, `%(objecttype)`, `%(objectsize)`, `%(objectmode)`, and `%(rest)`, including default format fallback when format is empty.
- Added `--batch-command` protocol handling for `info`, `contents`, and `flush`, with argument validation and `--buffer` / `--no-buffer` behavior.
- Kept missing-object output compatible with upstream-style batch responses and added minimal tree-ish/index object-expression support for batch mode lookups (`<treeish>:<path>` and `:<path>`).
- Added focused unit tests for batch format rendering and command parsing.
