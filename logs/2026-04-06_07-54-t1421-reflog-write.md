# t1421-reflog-write (2026-04-06)

## Goal
Make `tests/t1421-reflog-write.sh` fully pass and keep adjacent reflog/ref-store behavior stable.

## Root causes found

1. `test-tool -C <dir> ...` was not parsed in `grit test-tool` dispatch path, so helper calls failed with:
   - `error: test-tool: unknown subcommand '-C'`.
2. `reflog write` validation/diagnostics diverged from upstream:
   - unqualified refs were accepted,
   - OID error categorization for abbreviated vs nonexistent IDs did not match expected messages.
3. `reflog show --format=%gs` did not print subject-only message in this command path.
4. Reflog-write message normalization did not collapse embedded newlines/spaces to a single line.
5. Identity selection for `reflog write` did not match test-harness expectations around env/config precedence and timestamp source.

## Changes implemented

### `grit/src/main.rs`
- In `parse_cmd_args(...)`, added usage fallback line for `reflog write` parse failures:
  - `usage: git reflog write <refname> <old-oid> <new-oid> <message>`
- In `"test-tool"` dispatch:
  - added support for leading repeated `-C <dir>` pairs,
  - applies `set_current_dir` for each pair,
  - dispatches subcommands using post-`-C` argument slice.

### `grit/src/commands/reflog.rs`
- `run_show`:
  - implemented `%gs` special handling to print only reflog message subject (without OID/selector prefix).
- `run_write`:
  - stricter refname validation:
    - allow fully-qualified `refs/...` only when `check_refname_format` passes,
    - allow root refs via `is_root_ref_syntax`,
    - reject unqualified names with `invalid reference name: '<name>'`.
  - added OID parser with Git-compatible diagnostics:
    - abbreviated hex => `invalid old/new object ID`,
    - non-hex/nonexistent => `old/new object <raw|oid> does not exist`.
  - added object-existence verification using `repo.odb.exists`.
  - normalized reflog message via whitespace collapse (`split_whitespace().join(" ")`).
  - identity resolution aligned to tests:
    - committer env vars take precedence for name/email,
    - if committer env vars are unset, use author env vars; then config,
    - timestamp source:
      - uses `GIT_COMMITTER_DATE` when set,
      - when committer env identity is present and `.git/.test_tick` exists in test harness tree, uses `<tick> -0700`,
      - otherwise uses existing `GIT_AUTHOR_DATE` fallback or current time.
