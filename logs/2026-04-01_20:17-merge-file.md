# 2026-04-01 — grit merge-file

## Task
Implement `grit merge-file` — three-way file merge command, porting
`git/builtin/merge-file.c` and tests from `git/t/t6403-merge-file.sh`.

## What was built

### Library (`grit-lib/src/merge_file.rs`)
A full three-way line-level merge engine using the Myers algorithm
(via the `similar` crate):

- `merge(MergeInput) -> Result<MergeOutput>` — main entry point
- `MergeFavor` enum: None / Ours / Theirs / Union
- `ConflictStyle` enum: Merge / Diff3 / ZealousDiff3
- `is_binary(data)` — NUL-byte heuristic

**Algorithm:** annotates each base line with its change status in both
diffs, groups positions into hunks expanding to cover full op boundaries,
classifies as Unchanged / OnlyOurs / OnlyTheirs / Conflict.

Key edge cases handled:
- Changed op from one side overlapping equal region from other → Conflict
  (e.g. ours deletes lines 7-9, theirs modifies line 8 within that range)
- Pure insertions (zero old_range) attached to adjacent base position
- Union mode with missing trailing LF → inserts `\n` between sections
- Conflict markers starting mid-content get proper `\n` prefix

### CLI (`grit/src/commands/merge_file.rs`)
Flags: `-p/--stdout`, `-q/--quiet`, `--diff3`, `--zdiff3`, `--ours`,
`--theirs`, `--union`, `-L label` (up to 3), `--marker-size=N`.
Exit codes: 0 = clean, 1 = conflicts, 2+ = error.

Uses `std::process::exit()` to propagate the conflict exit code through
clap's error handling.

### Tests (`tests/t6403-merge-file.sh`)
22 tests ported from git's t6403, excluding `--object-id` (requires
ODB integration) and diff-algorithm (histogram not yet in similar usage):

- Basic merge (no changes, clean merge, subdirectory)
- Conflict detection and markers (standard and diff3)
- --ours / --theirs / --union resolution
- Custom labels (-L)
- Conflict in removed tail (delete vs change overlap)
- Binary file detection
- Marker size customization
- EOF without LF edge cases

## Test results
22/22 ported tests pass. Full harness: all 0 failures.

## Key design decisions
- Used op-boundary expansion rather than line-by-line status grouping
  to correctly handle the case where a Delete op from one side overlaps
  a Replace from the other
- `collect_new_lines` emits full replacement for Changed ops and
  proportional lines for Equal ops
