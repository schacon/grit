## Task

Complete `t1003-read-tree-prefix`.

## Root Cause

`read-tree --prefix` itself was behaving correctly, but the upstream harness sets
`GIT_INDEX_VERSION=4`. When `read-tree` rewrote the index, `grit-lib::Index`
emitted a header claiming version 4 while serializing plain v2/v3-style entries.
The next index read treated those bytes as v4 path-compressed entries, which
collapsed paths like `one` and `two/one` into a bogus `e` entry.

## Changes

- removed an unrelated duplicate `format_default_value` definition in
  `grit/src/commands/config.rs` so the branch would build again
- changed index serialization to downgrade version-4 requests to the newest
  format we actually support instead of writing a fake v4 header
- added a regression test covering a version-4-requested index round-trip
- removed an `unused_mut` warning in `grit/src/main.rs`

## Verification

- `cargo test -p grit-lib requested_v4_writes_a_compatible_index_format -- --nocapture`
- `cargo build --release -p grit-rs`
- `bash scripts/run-upstream-tests.sh t1003 2>&1 | tail -40`

Result: `t1003-read-tree-prefix` now passes `3/3`.
