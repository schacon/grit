### Root cause
- `grit apply` parsed `--whitespace=<action>` but did not use it in hunk matching/application.
- Context and remove lines required exact byte-for-byte matching, so whitespace-fix workflows could not fuzz-match equivalent whitespace.
- Added lines were written verbatim, so `--whitespace=fix` did not strip trailing whitespace in output.

### Fix implemented
- Updated `grit/src/commands/apply.rs`:
  - Added whitespace-canonical matching helpers (`canonicalize_ws_line`, `lines_equal`).
  - Extended `apply_hunks` signature to accept `whitespace_fix: bool`.
  - For `--whitespace=fix`:
    - Context/remove matching now tolerates whitespace-only differences.
    - Added/context output lines are normalized by trimming trailing space/tab.
  - Wired `whitespace_fix` through all apply/check call sites:
    - working tree apply
    - index apply
    - check mode
