## t1414-reflog-walk — make fully pass (12/12)

### Scope claimed
- Focused on `t1414-reflog-walk.sh` from the plan.
- Goal: move from partial pass to full pass.

### Root causes identified
1. Reflog merge commits were logged as `commit: ...` instead of `commit (merge): ...`, causing formatting mismatches in reflog-walk expectations.
2. Reflog identities for checkout/update-ref used wall-clock `now` rather than respecting `GIT_COMMITTER_DATE`/`GIT_AUTHOR_DATE`, which broke deterministic reflog ordering and age filtering used by tests.
3. `rev-list -g` implementation was too permissive and lacked:
   - proper requirement that a revision be provided in reflog-walk mode,
   - `--parents` output shape for reflog entries.

### Code changes made
- `grit/src/commands/commit.rs`
  - reflog message generation now emits `commit (merge): <subject>` for merge commits (`parents.len() > 1`), preserving existing initial/amend messages.

- `grit/src/commands/checkout.rs`
  - `resolve_checkout_identity()` now prefers `GIT_COMMITTER_DATE` and falls back to `GIT_AUTHOR_DATE` before using current time.
  - preserves existing name/email resolution behavior.

- `grit/src/commands/update_ref.rs`
  - introduced `resolve_reflog_identity(repo)` helper for update-ref reflog writes.
  - helper resolves name/email from env/config and uses `GIT_COMMITTER_DATE` (fallback `GIT_AUTHOR_DATE`) when present; otherwise uses current time.
  - replaced hard-coded `"grit <grit> 0 +0000"` identities in reflog writes with resolved identity string.

- `grit/src/commands/rev_list.rs`
  - refactored `-g/--reflog/--walk-reflogs` handling to parse options first, then run dedicated reflog-walk path.
  - added validation: `--walk-reflogs` now errors when no revision is provided.
  - added reflog-walk output mode support for `--parents` by parsing commit object and printing one-line `OID parent1 parent2 ...` format.

- `grit/src/commands/log.rs`
  - expanded reflog-walk behavior:
    - supports multiple reflog refs in one invocation (`HEAD main side`),
    - interleaves multiple reflogs by reflog-entry timestamp (descending), with command-line ref order as tie-breaker,
    - preserves natural newest-first order for single reflog walks,
    - supports reflog-specific filtering for:
      - `--no-merges` / `--merges`,
      - `--since` / `--since-as-filter` / `--until` using reflog entry timestamp,
      - pathspec filtering in reflog walk.
    - pathspec filtering for merge commits follows combined semantics (path must differ vs all parents), matching expected `log -g -- <path>` behavior in this test.
    - supports `%gd` unix-date selector rendering for `--date=unix`.
