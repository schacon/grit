# t2027-checkout-track

- Claimed and investigated on 2026-04-05 21:47 Europe/Berlin.
- Initial requested reproduction: `CARGO_TARGET_DIR=/tmp/grit-build-t2027 bash scripts/run-upstream-tests.sh t2027-checkout-track 2>&1 | tail -40` reported 4/5 passing, not the stale 2/5 shown in `plan.md`.
- Root cause:
  - `grit checkout trunk` fell through to pathspec handling when `trunk` matched multiple remote-tracking branches, so it never printed the expected `--track` hint.
  - `grit switch trunk` delegated to the system Git binary, but Apple Git 2.39.5 prints the older `git checkout --track` advice, which does not satisfy the upstream `git switch --track` expectation.
- Code changes:
  - Added ambiguous remote-tracking branch detection to `grit/src/commands/checkout.rs` before commit/path fallback.
  - Added the same ambiguity pre-check to `grit/src/commands/switch.rs` so the error path stays Git-compatible for this test instead of inheriting host-specific wording from the system Git binary.
- Validation:
  - `cargo build --release -p grit-rs`
  - `CARGO_TARGET_DIR=/tmp/grit-build-t2027 bash scripts/run-upstream-tests.sh t2027-checkout-track 2>&1 | tail -40`

- Result: `t2027-checkout-track` now passes 5/5.
