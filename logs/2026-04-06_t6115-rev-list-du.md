- `grit/src/commands/rev_list.rs`

    - `--disk-usage`
    - `--disk-usage=human`
    - `--use-bitmap-index` (accepted for compatibility)
    - `--unpacked` (accepted for compatibility)
  - Added strict invalid-value handling for `--disk-usage=<format>` with expected fatal message and exit code 128.
  - Implemented disk-usage reporting path:
    - collects selected commit/object IDs from rev-list result
    - computes per-object on-disk bytes from loose object file size when present
    - otherwise computes packed slot size via `.idx` offsets and `.pack` boundaries
    - prints either raw bytes or `<n> bytes` for `human` format.
- `grit/src/commands/cat_file.rs`
  - Added support for `%(objectsize:disk)` in batch formatting.
  - Implemented packed-object size cache from local indexes to support disk-size lookup.
  - Wired disk-size lookup into batch entry formatting so test pipelines using:
    - `git cat-file --batch-check="%(objectsize:disk)"`
    now return numeric values instead of literal placeholder text.
