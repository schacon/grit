# t4136-apply-check

## Goal

Make `tests/t4136-apply-check.sh` fully pass (6/6).

## Changes

- Added `validate_patch_semantics` in `grit/src/commands/apply.rs`:
  - Reject hunks with body lines but zero `+`/`-` lines (Git: `corrupt patch at line N`), unless `--recount`.
  - Reject `new file mode` combined with `copy from`/`to` or `rename from`/`to` (Git: `inconsistent header lines 2 and 3`).
  - Treat patch input where every file entry has no hunks as empty (`No valid patches in input`) unless `--allow-empty`.
