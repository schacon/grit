# t7521-ignored-mode

## Goal

Make `tests/t7521-ignored-mode.sh` pass (12/12) with grit.

## Root causes

1. **`git reset HEAD~1 --hard`**: clap `trailing_var_arg` put `--hard` / `-q` into `rest`, so `hard` was never set → mixed reset without worktree checkout. Tests that `clean` then `reset` left former tracked files as untracked and broke `status` expectations.
2. **`HEAD~1` pathspec fallback**: when ancestry resolution failed to classify as commit, first token was treated as pathspec → same silent skip of hard checkout.
3. **Porcelain v2 + ignored modes**: status needed `?`/`!` lines, optional `--ignored=` values, and Git-aligned untracked/ignored directory rules (matching vs traditional, normal vs all).

## Changes (summary)

- `grit/src/commands/status.rs`: ignored mode enum + collection rewrite; porcelain v2 untracked/ignored; `--ignored` optional string; reject matching+`-u no`.
- `grit/src/commands/reset.rs`: `normalize_reset_trailing_args`; ancestry-syntax hint in `split_commit_and_paths`; deterministic removal order + `remove_worktree_path_best_effort`.
- `grit-lib/src/rev_parse.rs`: `revision_spec_contains_ancestry_navigation`.
- `grit/src/commands/stash.rs`: `status::Args.ignored` type update.
- Harness: `data/test-files.csv`, dashboards after `./scripts/run-tests.sh t7521-ignored-mode.sh`.

## Validation

- `cargo test -p grit-lib --lib`
- `./scripts/run-tests.sh t7521-ignored-mode.sh` → 12/12
