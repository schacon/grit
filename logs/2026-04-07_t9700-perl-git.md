# t9700-perl-git

## Failures addressed

1. **Git.pm `config_bool` on invalid values** — Perl treats exit code 1 as “key missing”; invalid `--bool` must use a different exit (Git uses 128 / `fatal:`). `git config` now reports `fatal: bad boolean config value '…' for '…'` via `LibError::Message` and exits 128; typed int/bool-or-int get analogous fatals when a config key is known.

2. **`hash-object` for Git.pm** — Added `--no-filters` (no-op). Fixed `--stdin-paths` to read line-by-line so `hash_and_insert_object` does not block waiting for EOF while the bidi pipe stays open.

3. **`log .` pathspec** — Treat lone `.` and `..` as pathspecs during rev/path DWIM (Git.pm runs `log -n1 --pretty=format:%H .` from a subdirectory).

4. **Harness env** — `tests/test-lib.sh`: export `GITPERLLIB` defaulting to repo `git/perl` when unset; export test author/committer helper vars.
