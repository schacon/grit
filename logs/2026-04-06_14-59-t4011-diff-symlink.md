## t4011-diff-symlink

### Baseline
- `./scripts/run-tests.sh t4011-diff-symlink.sh` -> 4/8 passing.
- Initial failures:
  1. `diff-index -p -w` unsupported (`-w` parse failure).
  2. Symlink/content handling in `diff-index -p` used empty new-side data when index/worktree placeholders were present, yielding deletion hunks instead of replacements.
  3. `diff --no-index` read symlink paths with `fs::read`, failing with `No such file or directory` when symlink targets were missing.
  4. `diff` ignored `diff.<driver>.binary=true` for regular files (`file.bin`) and rendered textual patch output instead of `Binary files ... differ`.

### Implementation summary
#### `grit/src/commands/diff_index.rs`
- Added `-w` / `--ignore-all-space` option parsing and behavior:
  - new `ignore_all_space` flag in `Options`.
  - new `filter_entries_ignore_all_space` + `normalize_ignore_all_space`.
- Added symlink-aware worktree raw-content reading:
  - `read_worktree_path_raw` returns symlink target bytes for symlinks.
  - wired through `read_entry_raw_contents` and patch rendering path for placeholder worktree sides.
- Added binary-driver support in patch rendering:
  - `is_binary_driver_path` resolves `diff.<driver>.binary` through attributes/config.
  - binary forcing intentionally excluded for symlink modes so symlink diffs remain textual.
- Fixed config bool extraction (`ConfigSet::get_bool`) via `.and_then(Result::ok).unwrap_or(false)`.

#### `grit/src/commands/diff.rs`
- `--no-index` now reads paths via symlink-aware helper:
  - replaced direct `std::fs::read` with `read_path_raw`.
  - supports symlink target diff content even for dangling symlinks.
- Added binary-driver forcing in patch output path:
  - `binary_driver_for_path` checks `diff.<driver>.binary=true` using loaded attr rules/config.
  - integrated with existing binary detection in `write_patch_with_prefix`.
  - for binary output labels, `/dev/null` is preserved unprefixed while regular labels retain `a/`/`b/` prefix.
- Guarded driver-based binary forcing to non-symlink modes so symlink patches still show target text.
