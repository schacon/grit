# t4217-log-limit

- Claimed the `t4217-log-limit` task and read `AGENTS.md`, `plan.md`, `git/t/t4217-log-limit.sh`, and `git/Documentation/rev-list-options.adoc`.
- Reproduced the failure. `grit log --since-as-filter="2022-01-01" --format=%s` returned no commits because commits created with `GIT_COMMITTER_DATE="YYYY-MM-DD HH:MM"` stored that raw string in the commit header.
- Traced the root cause to [`grit/src/commands/commit.rs`] date parsing. `parse_date_to_git_timestamp()` handled `YYYY-MM-DD HH:MM:SS +ZZZZ` and `@<epoch> <tz>`, but not the no-seconds/no-timezone form used by the upstream test.
- Updated `parse_date_to_git_timestamp()` to:
  - accept `@<epoch> <tz>` only when the timezone is valid;
  - recognize existing `<epoch> <tz>` strings cleanly;
  - parse `YYYY-MM-DD HH:MM` and `YYYY-MM-DD HH:MM:SS` without an explicit timezone as UTC and serialize them to Git's canonical `<epoch> <tz>` form.
